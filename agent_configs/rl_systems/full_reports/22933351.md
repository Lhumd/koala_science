# Full report — Omni-fMRI: A Universal Atlas-Free fMRI Foundation Model

**Paper ID:** 22933351-73cd-4003-9032-4051bcde046c
**Reviewer:** rl_systems (claim-vs-evidence soundness)
**Verdict (Soundness): 2/10, confidence 4**

## Headline empirical claims

1. **"Universal atlas-free fMRI foundation model"** (title, abstract). Operational measure: pretraining on 49,497 sessions across 9 datasets; evaluation on 11 datasets / 16 tasks.
2. **"Omni-fMRI consistently outperforms existing foundation models"** (abstract). Tables 3, 4, 5, 6, Fig. 3, Fig. 4.
3. **"Data efficiency"**: 45.29% accuracy at 10% training data on HCP-task vs BrainMASS 23.12% (Fig. 3, Sec. B.3).
4. **"Methodological enabler"**: 32 hours on 4×A10G vs NeuroSTORM 13 days / BrainMASS 150h.
5. **"Atlas-free as principled solution"** (Sec. 5).

## Construct-validity audit

### "Universal"

Per Table 11 (pretraining dataset breakdown):

| Dataset | Subjects | Sessions |
|---|---:|---:|
| UKB | 38,301 | 38,301 |
| AOMIC | 348 | 348 |
| CHCP | 244 | 488 |
| ISYB | 130 | 130 |
| HCP-rest | 606 (70% of 875) | 2,022 |
| ABCD | (~70% of) 7,548 | (4 sess each) |
| ABIDE | … | … |
| HBN | 1,030 | 1,030 |
| PPMI | (70% of) … | … |
| **TOTAL** | 41,323 | 49,497 |

UKB ≈ 38,301 / 41,323 = **92.7%** of pretraining subjects.

The "universal" framing implies the model generalizes across vendors, protocols, populations, and acquisition setups. The demonstrated pretraining is overwhelmingly Siemens Skyra UK-population older-adult data with a small tail of younger / patient populations. No vendor-stratified, site-stratified, or population-stratified evaluation appears in the main text or appendices. The construct "universal" is operationalized only as "trained on more than one dataset".

### "Outperforms existing foundation models" — Type II/III mixing

Sec. 4.1 splits ABCD, ABIDE, HCP-rest, PPMI **70/30 between pretrain and downstream**. The split prevents same-subject leakage but preserves same-site, same-scanner, same-protocol exposure to the encoder. Tables 3 and 6 then report:

| Table 3 row | Dataset | Type | Same-scanner exposure during pretrain? |
|---|---|---|---|
| HCP sex | HCP | II | Yes (70%) |
| ABIDE age | ABIDE | II | Yes (70%) |
| ABCD sex | ABCD | II | Yes (70%) |
| PPMI PD | PPMI | II | Yes (70%) |
| ADNI(AD) | ADNI | III | No (held-out) |
| NKI age | NKI | III | No |
| SALD age | SALD | III | No |
| BHRC ASD | BHRC | III | No |

The headline tables mix Type II (in-pretraining) and Type III (held-out) results without flagging in the table. A reader interpreting Table 3 cannot distinguish between "Omni-fMRI generalizes well to held-out scanners/sites" and "Omni-fMRI's encoder is already tuned to the scanner/site distribution".

This is also load-bearing for the data-efficiency claim (Fig. 3): the 45.29% accuracy at 10% training on HCP-task is on a Type II target. The encoder has seen 70% of HCP-rest subjects (different sessions, but same scanner/site/protocol). The few-shot result therefore measures "few-shot finetuning of an already-in-distribution-pretrained model", a fundamentally weaker claim than "few-shot transfer to held-out distribution". The same data-efficiency curve on a Type III target (NKI, SALD, ADNI) would adjudicate this.

### "Atlas-free as principled solution"

The voxel-level Omni-fMRI is compared against atlas-based foundation models (BrainMASS, Brain-JEPA, BrainLM, BrainGNN) and atlas-free / voxel-level baselines (SwiFT, NeuroSTORM, TFF). It outperforms both groups. The "atlas-free is principled" framing implies the atlas vs no-atlas axis is the discriminator; but Omni-fMRI also beats voxel-level baselines (SwiFT, NeuroSTORM, TFF), so atlas-free is not the discriminating mechanism on those rows.

A direct controlled comparison — apply the Omni-fMRI architecture with ROI-aggregated (Schaefer-400) input, holding everything else constant — is missing. This would isolate the atlas-free contribution from architecture / scale / objective contributions.

## Reward / objective audit

Pretraining: scale-aware MAE (Eq. 4) on dynamic-patch tokenization. Four design components:
- (a) variance-based complexity gate for patching.
- (b) dual-path multi-scale embedding with ZeroMLP.
- (c) scale-aware MAE objective.
- (d) no per-patch normalization.

Ablations:
- (a) variance vs Shannon vs Laplacian vs MSE: Table 7. Isolated.
- (d) patch-norm vs no patch-norm: Table 8. Isolated.
- (b) ZeroMLP not ablated against e.g. simple resize-then-conv.
- (c) scale-aware MAE not ablated against uniform-weighted MAE.

Mechanism attribution within the architecture is therefore partial.

## Counter-hypothesis attempts

- **CH1: "Atlas-free wins" alternative — scale of pretraining data.** Omni-fMRI pretrains on 49,497 sessions; BrainMASS on 26 datasets ROI; Brain-JEPA on different scale. Conflated with representation choice. Not ruled out.
- **CH2: "Few-shot data efficiency" alternative — Type II contamination.** HCP-task is Type II. 70% of HCP-rest subjects are in pretraining. The encoder is in-distribution before few-shot finetuning. Not ruled out — paper does not run the same protocol on a Type III target.
- **CH3: "Universal" alternative — dominant single-source.** UKB = 92.7% of pretraining subjects. Model representations may primarily encode older-adult Siemens-Skyra UK-population characteristics. Not ruled out.

## Headline-number denominator audit

- Table 3 HCP sex: 95.54 (Omni-fMRI) vs 94.17 (NeuroSTORM, Brain-JEPA), +1.37 pp. No CIs cross-comparison; "* denotes large effect size with Cohen's d ≥ 0.8" — Omni-fMRI HCP entry has no asterisk. So this headline row is not a large-effect win, despite the table's visual emphasis.
- Table 6 ADNI(AD) LP F1: 83.65 vs SwiFT 74.70 / NeuroSTORM 79.43. Real win but on Type III with n=497 subjects.

## All weaknesses (Soundness lens, severity)

- **High:** Construct-coverage gap on "universal" — 92.7% pretraining from UKB, no vendor/protocol/site stratification. Verbatim-anchored to Table 11 and abstract framing.
- **High:** Type II / Type III headline mixing without flag in Tables 3, 6; data-efficiency claim load-bears on Type II.
- **Medium:** Within-architecture atlas-vs-no-atlas ablation absent; "atlas-free as principled" is a within-claim attribution failure.
- **Medium:** Mechanism-attribution within architecture is partial — components (b) and (c) not ablated isolated.
- **Low:** Compute-efficiency comparison cross-tier (4×A10G vs 4×A6000 vs 8×V100) without normalized FLOPs/throughput.

## Theoretical context (out of Soundness scope)

The dynamic-patch / variance-gate construction (Sec. 3.2) is mathematically clean. ZeroMLP for non-uniform patch sizes is a reasonable architectural choice. No proof correctness issues observed within Soundness scope.

## Bottom line

Score 2 / confidence 4. Score-2 triggers fire on two load-bearing claims (universal, Type II/III mixing). Per CALIBRATION PRINCIPLE #2, these triggers fire regardless of offsetting strength elsewhere. The paper's well-executed sections (variance ablation Table 7, patch-norm ablation Table 8, scaling-law experiments) belong in the comment as caveats but do not rescue the headline claims.
