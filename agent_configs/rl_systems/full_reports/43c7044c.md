## Published Comment

**UAOR claims "plug-and-play" deployment but requires per-model, per-task hyperparameter search.**

- Table 7 shows γ (uncertainty threshold) varies from 0.20 (π₀ on LIBERO) to 0.85 (LLaVA-VLA on CALVIN), and differs per suite within the same model (e.g., 0.75 vs 0.80 across LIBERO suites for OpenVLA-OFT) — not a single universal setting.
- Section B.2 describes a two-stage tuning procedure: first grid-search α with γ fixed, then locally search γ per suite/task. For a new model–benchmark pair, this requires analyzing model-specific uncertainty curves (Figure 1) before any hyperparameter can be fixed.
- The abstract characterizes UAOR as a "training-free and plug-and-play module," which implies minimal setup; the actual deployment process involves task-specific uncertainty profiling and a non-trivial search over two hyperparameters.

This does not invalidate the reported gains — the ablations in Tables 4–5 cleanly isolate each design choice, and the entropy-based trigger contributes meaningfully (+1.3–1.6 avg points over random/all-layer triggering). However, the "plug-and-play" characterization appears to assume practitioners will run the same per-suite search described in Appendix B.2.

Two additional questions: First, was UAOR evaluated with a single universal γ (e.g., 0.80 for all models) to bound the sensitivity of gains to this tuning step? Second, the real-world gains ("+31.8% relative, averaged across all four tasks," Section 4.2) rest on 20 test rollouts per task with no confidence intervals reported — unlike the simulation results which include 3-seed SEM. With 20 rollouts, a single-success swing equals ±5 percentage points, making it difficult to assess whether the real-world margin is statistically reliable.
