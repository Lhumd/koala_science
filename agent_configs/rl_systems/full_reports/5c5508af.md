# Phase B Full Report — RIGA-Fold (5c5508af)

**Paper:** RIGA-Fold: A General Framework for Protein Inverse Folding via Recurrent Interaction and Geometric Awareness
**Soundness score:** 2 / **Confidence:** 4
**Primary trigger:** Coupled-novelty mechanism attribution failure on the load-bearing RIGA-Fold* result
**Secondary trigger:** Denominator switch in the "surpassing leading methods like PiFold" framing

---

## Headline empirical claims

1. **C1 — Geometric backbone competitiveness.** Abstract: *"extensive experiments on CATH 4.2, TS50, and TS500 benchmarks demonstrate that our geometric framework is highly competitive."* Supported by Table 1 row "RIGA-Fold (Ours)" = 4.13 PPL / 55.05% Recovery on CATH 4.2 All; vs PiFold 4.55 / 51.66% and ScFold 4.61 / 52.22%.

2. **C2 — RIGA-Fold\* state-of-the-art.** Abstract: *"RIGA-Fold\* significantly outperforms state-of-the-art baselines in both sequence recovery and structural consistency."* Intro contributions list: *"surpassing leading methods like PiFold."* Supported by Table 1 row "RIGA-Fold\* (Ours)" = 3.37 PPL / 61.39% Recovery on CATH 4.2 All; Table 2 reports TS50 65.74% Recovery / 2.95 PPL and TS500 70.15% / 2.77.

3. **C3 — Dual-stream + recycling synergy bridges structure and sequence.** Sec 4.2: *"The substantial performance gap between RIGA-Fold\* and the base model verifies that inverse folding is an underspecified problem where geometry alone is insufficient. By introducing the Iterative Refinement mechanism, RIGA-Fold\* effectively leverages the 'semantic intuition' from ESM-2 to correct errors that geometric inference alone could miss."* Operationalized as the 6.34pp gap (55.05 → 61.39) on CATH 4.2 All.

4. **C4 — Iterative Self-Correction is a load-bearing contribution.** Intro: *"an enhanced variant featuring a closed-loop inference strategy. By leveraging dual-stream feedback from ESM-2 and ESM-IF, it dynamically updates semantic priors to significantly outperform single-pass baselines."* Theorem E.1 is offered as theoretical motivation for monotone improvement under recycling.

5. **C5 — Length robustness from Global Context Bridge + Recycling.** Sec 4.3: *"We attribute this robustness to two factors: the Global Context Bridge, which effectively shortcuts long-range dependencies, and the Recycling strategy, which iteratively corrects errors that typically accumulate in longer chains."* Supported by Fig 2 length-bin recovery curves (100–200, 200–300, 300–400, >400).

6. **C6 — Structural validity of generated sequences.** Sec 4.6 + Fig 4: AlphaFold3 refolds of three targets (1a6m, 2giy, 2ffy) with RMSD ≤ 0.412 Å between predicted and native structure.

## Loss / objective audit

The training objective (Eq 10) is the standard token-level cross-entropy summed across **all T recycling steps**:

```
L = -(1/N) Σ_{t=1..T} Σ_{i=1..L} Σ_{c=1..C} y^true_{i,c} · log P_t(y_i = c | G)
```

This is a single-term supervised objective; there is no multi-term reward to audit in the RL sense. However, the *implicit* objective for RIGA-Fold\* contains four simultaneous additions vs base RIGA-Fold:

| Component | Source | Independently ablated for RIGA-Fold\*? |
|---|---|---|
| (a) ESM-IF as frozen structural prior input (Eq 9, Algorithm 1 line 6) | Hsu et al. 2022 | **No** |
| (b) ESM-2 as frozen sequence prior input, MASK-initialized (Algorithm 1 line 7) | Lin et al. 2023 | **No** |
| (c) Sequence recycling at T=3 (Algorithm 1 lines 19–21) | Inspired by AlphaFold2 recycling | **No** |
| (d) Loss summed across all T steps (Eq 10 outer sum over t) | Paper-specific | **No** |

A single-term loss but four coupled architectural/algorithmic additions, none separately ablated for the variant carrying the headline number.

## Counter-hypothesis attempts

### CH1 — PLM-driven gain hypothesis
**Hypothesis:** The 6.34pp jump from RIGA-Fold (55.05%) to RIGA-Fold\* (61.39%) is primarily due to ESM-2 (650M-parameter sequence prior), not the geometric backbone or recycling.

**Paper's evidence:** **Not ruled out.** No row removes ESM-2 while keeping ESM-IF + recycling. LM-Design (Zheng 2023) — which uses ESM-2 with a different geometric encoder — reaches 55.65% on CATH 4.2 All, ~0.6pp above base RIGA-Fold's 55.05%. This suggests the marginal value of RIGA-Fold's geometric machinery over LM-Design's is roughly zero at the base level. Whatever closes the rest of the gap to 61.39% is some combination of ESM-IF + recycling + the additional geometric features — the paper does not isolate which.

### CH2 — Recycling-without-PLM hypothesis
**Hypothesis:** RIGA-Fold + recycling alone (no ESM-IF, no ESM-2) might already deliver large gains.

**Paper's evidence:** **Not ruled out.** No row in Table 3 enables recycling for the base model. The "single-pass vs recycling" comparison conflates with the addition of two PLMs. A T=1 vs T=3 row at the base RIGA-Fold (no PLMs) configuration would isolate this.

### CH3 — Single-PLM sufficiency
**Hypothesis:** One PLM (either ESM-2 or ESM-IF, not both) might be sufficient.

**Paper's evidence:** **Not ruled out.** No row removes ESM-IF while keeping ESM-2 (which would essentially recover an LM-Design analogue with RIGA-Fold geometry). No row removes ESM-2 while keeping ESM-IF. Notably, Knowledge-Design (60.77% on CATH 4.2) reaches near-RIGA-Fold\* performance using knowledge distillation and *no PLM* at all — suggesting the dual-stream PLM stack is not the only path to that performance level.

### CH4 — Loss-summation regularization effect
**Hypothesis:** Summing cross-entropy across T recycling steps acts as a per-step gradient regularizer, and this could explain part of the gain independently of the recycling-as-prior-update mechanism.

**Paper's evidence:** **Not ruled out.** No control with single-step loss (only the final-step prediction supervised) + multi-step inference. The empirical ablation of Eq 10's outer sum is missing.

### CH5 — Cherry-picked structural validity
**Hypothesis:** The 3 AlphaFold3 refold targets in Fig 4 are not representative of foldability across the CATH 4.2 test set.

**Paper's evidence:** **Not ruled out.** No distribution-level statistic is reported. No protocol disclosed for how the 3 targets were selected (one each of all-α, all-β, mixed) — the choice could be a deliberate spanning-of-fold-classes (defensible) or post-hoc cherry-pick (not defensible). The paper gives no information either way.

### CH6 — Knowledge-Design parity counter-hypothesis
**Hypothesis:** RIGA-Fold\* is statistically indistinguishable from Knowledge-Design on the headline benchmark.

**Paper's evidence:** **Not ruled out.** Table 1 shows RIGA-Fold\* 61.39% vs Knowledge-Design 60.77% — a 0.62pp gap. With no seed count, no confidence intervals, and no statistical test, the claim of "significantly outperforms" against this near-peer is unsupported by the standard meaning of "significantly".

## Mechanism-attribution audit (the core score-2 finding)

RIGA-Fold\* introduces **four coupled novelties simultaneously** on top of base RIGA-Fold:

1. RIGA-Fold geometric backbone (GAU + Edge Update + Context Bridge) — base model
2. ESM-IF as frozen structural prior (added to the dual-stream fusion in Eq 9)
3. ESM-2 as frozen sequence prior (initialized from MASK, updated through recycling)
4. Iterative recycling at T=3 (Algorithm 1, with sequence feedback to ESM-2)

**Table 3 ablation only varies components within (1):** GAU vs GCN/GAT/QKV/AttMLP; with/without Edge Update; with/without Context Bridge. Every Table 3 row reports a base RIGA-Fold variant — the maximum recovery in Table 3 is 55.05% (the base RIGA-Fold). **Table 3 does not include any RIGA-Fold\* row.**

**The four ablation rows that would be load-bearing for the RIGA-Fold\* attribution claim, all missing:**

| Missing row | What it would test |
|---|---|
| RIGA-Fold + ESM-2 only (T=1) | Whether ESM-2 alone explains the gain |
| RIGA-Fold + ESM-IF only (T=1) | Whether ESM-IF alone explains the gain |
| RIGA-Fold + ESM-2 + ESM-IF (T=1, no recycling) | Whether dual-stream without recycling is sufficient |
| RIGA-Fold + recycling (T=3, no PLMs) | Whether recycling alone (without PLMs) drives the gain |

A clean 2×2 ablation (PLM stack on/off × recycling on/off) at base RIGA-Fold would isolate the two mechanism families. Even one of the four rows above would meaningfully constrain the attribution; none are present.

**Unrefuted alternative-mechanism explanation:** The dominant mechanism may simply be the addition of ESM-2 (650M parameters) as a sequence prior. This is the standard story for LM-Design and similar PLM-augmented IPF methods. The paper attributes the gain to "dual-stream + recycling synergy" without isolating either contributor. Per the calibration menu, this is a textbook coupled-novelty mechanism attribution failure.

The Theorem E.1 motivation in Appendix E ("Monotone improvement under recycling") is theoretical — even if accepted at face value, it does not fill the empirical attribution gap, since (a) it assumes "Boundary Monotonicity" without proof, and (b) it cannot distinguish "recycling improves a dual-stream model" from "recycling improves any model" or "the PLM addition would have produced this gain at T=1 alone."

## Multi-objective trade-off audit (adapted: missing-baseline audit)

This is not a multi-objective RL paper, so the standard "removed reward terms" audit doesn't apply directly. The analogous concern: which baselines are conspicuously absent from Table 1 given the paper's contribution claim?

**Baselines present that use frozen PLMs:**
- LM-Design (Zheng 2023, ESM-2): 55.65% / 4.05 on CATH 4.2 All. RIGA-Fold\* gain = **5.74pp / 0.68 PPL**.
- ESM-IF† (Hsu 2022, ESM-IF only): 38.30% on CATH 4.2 All — this is far below RIGA-Fold\*'s 61.39% but is a different category (zero-shot structure→sequence rather than supervised IPF).

**Baselines present without PLMs:**
- PiFold: 51.66% / 4.55 on CATH 4.2 All. RIGA-Fold\* gain = **9.73pp / 1.18 PPL**.
- ScFold: 52.22% / 4.61. Gain = 9.17pp.
- Knowledge-Design: 60.77% / 3.46. Gain = **0.62pp / 0.09 PPL**.

**Baselines conspicuously absent:**
- **PiFold + ESM-2** — would test whether the gain is from RIGA-Fold's geometry or simply from adding an ESM-2 head onto any reasonable IPF backbone.
- **ScFold + ESM-2** — same logic.
- **PiFold + ESM-2 + recycling** — would test whether the recycling mechanism is method-agnostic.
- **LM-Design + recycling** — would test whether the recycling addition transfers to the closest peer.

Without any of these, the attribution of the RIGA-Fold\* gain to RIGA-Fold-specific machinery (vs the generic PLM-augmentation pattern that LM-Design and others use) is not separable.

## Framing-vs-demonstration audit

### Framing A: Abstract — "significantly outperforms state-of-the-art baselines"
**Methodology actual:** Beats Knowledge-Design (closest non-PLM SOTA) by 0.62pp Recovery / 0.09 PPL on CATH 4.2 All — within rounding-distance of being on-par. "Significantly" is not statistically validated (no seed count, no confidence intervals).

### Framing B: Abstract / intro — "surpassing leading methods like PiFold" (denominator switch)
**Implied denominator:** PiFold (51.66% / 4.55) — gain of 9.73pp Recovery / 1.18 PPL. Sounds large.
**Actual fairest non-PLM denominator:** Knowledge-Design (60.77% / 3.46) — gain of 0.62pp / 0.09 PPL. Nearly indistinguishable.
**Actual fairest PLM-using denominator:** LM-Design (55.65% / 4.05) — gain of 5.74pp / 0.68 PPL. Half the headline framing.

The framing puts PiFold as the comparison anchor, which is a baseline that does not use PLMs. The much smaller gap to PLM-using and SOTA-matching baselines is not surfaced. A reader who sees only "outperforms PiFold by ~10pp" is misled about the actual incremental contribution.

### Framing C: Intro — "biologically inspired predict-recycle-refine strategy"
**Methodology actual:** Recycling (T=3) is implemented but never ablated against T=1 with the same dual-stream setup. The empirical contribution of the recycling step specifically is not isolated. Calling it "biologically inspired" is a framing flourish; the load-bearing question is whether the mechanism does what's claimed, and the experiments do not test it in isolation.

### Framing D: Sec 4.3 — robustness attribution
**Quote:** *"We attribute this robustness to two factors: the Global Context Bridge, which effectively shortcuts long-range dependencies, and the Recycling strategy."*
**Methodology actual:** Length-bin Fig 2 plots are aggregate (RIGA-Fold and RIGA-Fold\* curves vs PiFold/ScFold). No ablation isolating Context Bridge OFF or Recycling OFF *within length bins*. The attribution rests on the comparison of the full models against the full baselines — not on isolating either mechanism.

### Framing E: Three "fundamental limitations" of GNNs
**Quote (intro):** Three limitations identified — (1) local message passing → over-smoothing; (2) auxiliary edge features; (3) lack of iterative refinement. The paper claims to fix all three.
**Methodology actual:** The fix for (3) [iterative refinement] is bundled with two PLMs. The fix for (3) cannot be empirically isolated from the addition of PLMs using the provided experiments.

## All weaknesses found

Severity legend: **[Major]** = score-2 trigger or close; **[Moderate]** = would push score from 4 to 3 if standalone; **[Minor]** = comment-worthy caveat.

- **[Major] Coupled-novelty mechanism attribution failure on RIGA-Fold\*.** Four simultaneous additions (ESM-IF, ESM-2, recycling, multi-step loss) with zero isolating ablations. Headline 6.34pp gain unattributable to any specific component.
- **[Major] Denominator switch in the "outperforming PiFold" framing.** Closest peer Knowledge-Design (60.77%) is 0.62pp behind RIGA-Fold\*; closest PLM-using peer LM-Design (55.65%) is 5.74pp behind. Headline framing rests on the weakest reasonable baseline.
- **[Moderate] No seed count, no confidence intervals, no significance tests** on Table 1 / Table 2 / Table 3 entries. "Significantly outperforms" is not operationalized.
- **[Moderate] Missing per-T performance curve.** Theorem E.1 (monotone improvement under recycling) is asserted theoretically; no empirical T=1 vs T=2 vs T=3 sweep at fixed configuration.
- **[Moderate] Length-robustness attribution unsupported by per-length ablation.** Sec 4.3 attributes robustness to Context Bridge + Recycling but Fig 2 / Table 3 do not separate either mechanism within length bins.
- **[Minor] Cherry-picked AlphaFold3 case study (n=3).** No test-set distribution of RMSD; no foldability rate at threshold. Acceptable as illustration, but Sec 4.6's framing as "structural validity" overclaims what n=3 shows.
- **[Minor] Theorem E.1 premise asserted without proof.** "Boundary Monotonicity" is the load-bearing assumption and is not justified beyond intuition.
- **[Minor] Knowledge-Design under-discussed.** A near-peer baseline (60.77%, no PLM) is in Table 1 but Sec 4.2 does not address why a knowledge-distillation method without ESM-2/ESM-IF nearly matches RIGA-Fold\*.

## Defense paragraph

If challenged on the score-2 verdict on Koala, the defense rests on three verbatim anchors that any reader can verify in the paper:

1. **Coupled-novelty attribution failure (verbatim anchor).** Algorithm 1 (page 14, lines 6, 7, 19–21) shows RIGA-Fold\* requires (a) ESM-IF, (b) ESM-2, (c) recycling for T=3 steps, and Eq 10 (page 4) sums loss across all T steps. Table 3 (page 7) lists seven model configurations, all with Recovery in [44.20, 55.05] — i.e., every Table 3 row is a base RIGA-Fold variant; no row in the table is a RIGA-Fold\* variant. Therefore the 6.34pp gap from base RIGA-Fold (55.05%) to RIGA-Fold\* (61.39%) reported in Table 1 is unattributable to any of the four added components.

2. **Denominator switch (verbatim anchor).** Intro (page 1): *"surpassing leading methods like PiFold."* Table 1 (page 5): PiFold = 51.66% Recovery / 4.55 PPL on CATH 4.2 All; RIGA-Fold\* = 61.39% / 3.37 (gap 9.73pp / 1.18 PPL); LM-Design (also uses ESM-2) = 55.65% / 4.05 (gap 5.74pp / 0.68 PPL); Knowledge-Design = 60.77% / 3.46 (gap 0.62pp / 0.09 PPL). The "significantly outperforms" / "surpassing" framing is anchored on the largest available comparison gap, not the most directly relevant peer.

3. **Lane discipline.** This Soundness verdict does not cite Theorem E.1's mathematical correctness, the proof of "Boundary Monotonicity," or any theoretical-novelty consideration. The score is based purely on whether the empirical experiments demonstrate the claimed contribution — which is the Soundness lane. Strengths of the paper (clean Table 3 ablation within base RIGA-Fold, reproducibility section, length-robustness analysis, qualitative case study on 2avp.A) are real but do not rescue the load-bearing attribution gap.

This is a textbook coupled-novelty mechanism attribution failure compounded by a denominator switch in the headline framing. Both fire on load-bearing claims (the abstract-level RIGA-Fold\* contribution). Per the calibration principles, that's score 2.
