# Scratch: VLM-Guided Experience Replay / VLM-RB (c39d243f)

## Load-bearing claim inventory
1. Abstract: "agents trained with our proposed prioritization method achieve 11–52% higher average success rates" 
2. Abstract: "improve sample efficiency by 19–45% compared to previous approaches"
3. Sec 4.3: "+241.7% improvement over UER" and "+70.8% over PER" on DoorKey-16×16 (relative ASR)

## Operational metric audit
- Metric: Average Success Rate (ASR) = fraction of episodes where goal reached within T_max steps, averaged over N=32 eval episodes per seed, M=5 seeds. Final = ASR averaged across M seeds with SEM.
- "11-52% higher average success rates": Table 1 shows individual improvements ranging 0% to 241.7%. The "11-52%" range is unclear — individual task relative gains range from 0% to 241.7% (UER) or 0% to 70.8% (PER). Absolute gains in some challenging settings (16x16 DQN/UER: 0.79-0.36=0.43pp) could yield ~11-52pp absolute improvements. Abstract claim possibly refers to absolute improvement range in pp — but Table 1 reports RELATIVE improvements. This creates mild ambiguity about what "11-52%" refers to.

## Observation space mismatch (key construct gap)
- Construct claimed: "VLM-RB leverages pre-trained visual semantics to guide prioritization"
- Agent observation: symbolic state s_t ∈ {0,...}^{N×N×3} (DoorKey) or state vector s_t ∈ ℝ^{40} (OGBench/Scene) — agents NEVER observe pixels
- VLM operates on: rendered frames ψ(s_t) — visual frames that agents never see
- The "semantic grounding" is from visual interpretation of rendered scenes, not from the agent's actual observation space
- Modified Game test (Fig 4): when rendered frames are visually "Misleading" or "Abstract", performance drops to/below PER level — demonstrating the method's gain IS contingent on visual coherence of rendered frames
- **The paper doesn't evaluate VLM-RB on pixel-based RL**, where VLM and agent share the same observation modality — this would be the natural setting for "VLM-guided" prioritization
- This is acknowledged in Limitations: "VLM-RB does not apply to domains where underlying state lacks visual representation" — but the evaluated domains (state-based symbolic obs) don't really have visual representation for the AGENT either

## Reward/objective audit
- DoorKey: zero reward until terminal goal (sparse), linearly decaying at success
- OGBench/Scene: potential-based shaping r_t = -1 + γφ(s_{t+1}) - φ(s_t) where φ(s) = fraction of satisfied subgoals
- VLM prompt (DoorKey): "Does this clip contain a clear instance of goal satisfaction anywhere in it? ... Output exactly Answer: Yes or Answer: No."
- VLM scoring is binary (0/1), not graded — all "success visible" clips score equally
- For OGBench: combined VLM × TD-error priority, allowing fine-grained sorting within success/failure bins

## Ablations and mechanism
- Modified Game (Fig 4): isolates visual semantics contribution — well-designed ablation
- λ_max sensitivity (Table 4 / Appendix F): mixing schedule is load-bearing (pure prioritized sampling hurts)
- VLM model size (Appendix F.2): scaling to 3B/8B doesn't consistently improve RL performance
- No ablation on prompt specificity (task-specific prompts vs generic) — gap for generalization claim

## Baseline strength
- UER and PER are appropriate standard baselines
- Also compared against AER, ERO, ReLo — all fail on challenging tasks (Fig 5)
- Baselines use same backbone architecture, optimizer, training schedule per Appendix E — fair

## Seed count and statistical reporting
- M=5 seeds, SEM shown as shaded regions — meets field norm
- Table 1 shows best ASR and sample efficiency per configuration

## Counter-hypotheses
1. Does the gain come from the observation-space gap (VLM sees more than agent) rather than semantic grounding? Partially addressed by Modified Game test — corrupting rendered frames breaks gains.
2. Is the warm-start (10 demos for OGBench) doing the work? Consistent across all methods, so it's controlled.
3. Is the annealing schedule (λ_0=0→λ_max=0.5) the key? Table 4/Appendix F.1 ablates λ_max — yes, retaining uniform component is essential; but VLM still outperforms PER even with optimal λ.
