## Construct Inventory

### Claim 1 — "Adaptive efficiency" / "75% less compute"

**Construct claimed:** Computational efficiency — the method uses significantly less compute while achieving equal or better task success.

**Operational metric:** Number of Function Evaluations (NFE) = number of gradient steps taken by the optimizer (average over rollouts). Success rate on LIBERO benchmarks.

**Specific unmeasured aspects:**
1. **Wall-clock time per inference step is not measured.** Each GeCO step is a gradient computation through the network (backward pass through the neural field to get f_θ(a,s) and then a gradient update step), while each Rectified Flow step is a single forward pass. GeCO requires at least one additional backward pass per NFE step beyond what a forward-only baseline requires. The paper reports "75% less compute" purely in terms of NFE count (5 vs 20), but never discloses actual wall-clock latency or FLOPs per step. A backward pass through f_θ is substantially more expensive than a forward pass — typically 2-3× the cost. If each GeCO step costs ~3× what a Rectified Flow step costs (due to the gradient computation required for the optimization update), the true compute savings may be zero or even negative.
2. **Memory overhead of backward-pass-based inference is not measured.** Gradient-based optimization through the network at inference time requires retaining activations for backprop, increasing peak memory usage significantly compared to pure forward-pass inference. This is never disclosed.
3. The "75% less compute" framing in the abstract implies a comparison against the same computational primitive (step cost), but GeCO steps are not the same cost as flow integration steps.

### Claim 2 — "Intrinsic OOD detection" / "training-free OOD detector"

**Construct claimed:** The field norm after optimization convergence reliably distinguishes in-distribution (ID) from out-of-distribution (OOD) robotic states without any additional training.

**Operational metric:** Score(s_t) = ‖f_θ(â(s_t), s_t)‖ (final residual gradient norm), evaluated on LIBERO-Goal (ID) vs LIBERO-Spatial (OOD). AUROC = 0.93, TNR = 82.4% at TPR ≈ 90%.

**Specific unmeasured aspects:**
1. **OOD construct is suite-level distribution shift (task layout), not genuine semantic OOD.** The paper's OOD scenario is training on LIBERO-Goal and testing on LIBERO-Spatial — both benchmarks use the same workspace, same object vocabulary, and same manipulation skills; the shift is only in spatial arrangements and object placement. Genuine OOD for deployment would include novel object categories, extreme lighting/visual corruption, gripper slippage, physical perturbations, or entirely unseen manipulation primitives. The paper never evaluates whether the OOD signal generalizes beyond this narrow suite-level shift.
2. **The binary OOD threshold τ_OOD is not evaluated for sensitivity.** The paper selects a threshold at TPR ≈ 90%, but does not report how the detector performs across different ID/OOD dataset pairs, nor does it show whether the same threshold generalizes across task distributions. A detector that requires per-deployment calibration is not truly "training-free" in the broad sense.
3. **TNR = 82.4% at TPR = 90% (Moving Average) means 17.6% false positive rate.** In continuous robot operation, 17.6% of in-distribution planning calls would be flagged as OOD and episode terminated. The paper reports "Time Saved = 42.4%" but does not report the cost of false episode terminations on ID tasks (i.e., how often the robot incorrectly aborts a valid task due to false OOD flags). This operational cost is entirely unmeasured.

### Claim 3 — "Better drop-in replacement for VLA action heads"

**Construct claimed:** GeCO is a broadly superior drop-in replacement across VLA architectures and benchmarks.

**Operational metric:** Task success rate on VLABench, RoboTwin 2.0, and LIBERO simulation benchmarks.

**Specific unmeasured aspects:**
1. **The VLA integration fixes time t=0 during fine-tuning** (Sec. 4.2: "we adapt it to our time-unconditional framework by simply fixing the input time t=0 during fine-tuning"). This means the π0 backbone is not re-trained from scratch for time-unconditioned operation; it is fine-tuned with a frozen time input. Whether the backbone's internal representations (which were learned under time-conditional dynamics) are compatible with stationary field learning is not ablated. A π0 model truly designed for GeCO from scratch may behave differently.
2. **Table 4 shows GeCO (π0) averages 93.9 vs π0 baseline 94.2** — the comparison is actually slightly *below* baseline for π0, although close. The framing of "better drop-in replacement" overstates the LIBERO π0 result. The π0.5 result is stronger (96.6 vs 96.9, GeCO wins), but the headline claim of general superiority is not uniformly supported across all VLA-benchmark combinations.

## Reward/Objective Inventory

No RL reward function — this is an imitation learning paper. The training objective is the flow-matching regression loss:

**GeCO Loss (Eq. 3):** L(θ) = E_{(s_t,a)} E_{ε,γ} ‖f_θ(x_γ, s_t) − g*(a, ε, γ)‖²

where:
- x_γ = γa + (1−γ)ε (noisy interpolation)
- g*(a, ε, γ) = (ε − a)c(γ) (restoring direction toward clean action)
- c(γ) = piecewise decay function with scale λ=4 and onset a_0=0.1

**Baseline (Rectified Flow) Loss:** Standard flow matching with time-conditioned v_θ(x_γ, γ, s_t), regressing toward (ε − a) (the constant velocity field).

**Key difference:** GeCO does not provide γ to the model at inference — γ is only an implicit training variable. This forces f_θ to learn a single stationary field consistent across all noise levels.

**Observation:** Both methods use the same backbone (DiT-1D transformer), same dataset. The differences are: (1) GeCO does not receive γ as input; (2) GeCO uses adaptive gradient-based inference; (3) GeCO uses scaled restoring direction c(γ) instead of constant (ε-a). There is no RL reward or auxiliary objective.

## Counter-Hypothesis List

**CH1 (backward-pass cost concealment):** The "75% less compute" claim (5 NFE vs. 20 NFE, Table 1) is stated in terms of forward-pass equivalent NFE, but GeCO's inference loop (Algorithm 1, lines 5-9) requires computing f_θ(a^(k), s) and then performing a gradient update a^(k+1) = a^(k) − η·v_k where v_k = f_θ(a^(k), s). Computing the gradient update direction is itself a forward pass through f_θ. A "20-step Rectified Flow" runs 20 forward passes; a "5-step GeCO" runs 5 evaluations of f_θ(a^(k), s). But to compute the gradient-based update a^(k+1), you also need the gradient ∇_a f_θ if the optimization requires it — HOWEVER, re-reading the algorithm: v_k = f_θ(a^(k), s) directly is the update direction (gradient descent on the objective using f_θ as the gradient). So GeCO uses f_θ's output (not ∇_a f_θ) as the update direction. This means the NFE count is a fair comparator — 5 forward passes vs 20 forward passes. The "75% less compute" claim in terms of NFE IS fair. However, wall-clock per NFE is still potentially different since GeCO uses gradient-descent optimization dynamics vs ODE integration — the step computation may differ due to the step-size schedule ({η_k}^30_{k=1}) being adaptive vs fixed ODE integration. The paper still does not report absolute wall-clock time.

**CH2 (early stopping confound — simpler tasks, not better policy):** GeCO exits early on "simple" states due to convergence. But "simple" states are ones where any reasonable action sequence suffices — the improvement in NFE might simply reflect that GeCO converges trivially on easy tasks rather than that the policy is more efficient. The hard tasks (Blocks Ranking in RoboTwin 2.0) show near-zero success for all methods including GeCO (Table 2), suggesting the method's limitations appear precisely where the adaptive mechanism would need to invest maximum compute. This is not directly a score-2 trigger since the paper acknowledges "Blocks Ranking (Size) remains challenging for all methods."

**CH3 (OOD is in-distribution relative to visual features):** LIBERO-Goal and LIBERO-Spatial use the same objects, same robot, same visual style. The "OOD shift" is purely spatial arrangement. A detector that works on this may fail on genuine visual OOD (new object categories, lighting conditions, physical perturbations). The paper does not test this — and the OOD claim in the abstract ("remaining low for in-distribution states while significantly increasing for anomalies") implies broader generality than what is demonstrated.

**CH4 (real-world baseline weakness):** In Table 7, the baseline G0 Plus achieves only 10% (Nut Assembly) and 20% (Tube Arrangement). These are strikingly low success rates for a pre-trained VLA model on simple manipulation tasks. The paper attributes GeCO's gains (70% and 80%) to "enhanced sample efficiency and superior generalization." But if the baseline is performing poorly due to inadequate fine-tuning data (4 epochs of fine-tuning with batch 32 = very few gradient steps), then the gains might reflect that GeCO's adaptive optimization is better at extracting useful gradients from undertrained models, not that the method would show the same advantage with a properly trained baseline.

## Multi-Objective Trade-Off Audit

The paper does not use a multi-term reward — it is an imitation learning setting. No multi-objective trade-offs apply in the RL sense. The adaptive computation trade-off is: GeCO converges early on easy states (fewer NFE) but applies maximum budget to hard states. The paper shows the NFE distribution (Figure 2, Figure 6) and demonstrates this empirically. This is an internal trade-off between per-state compute and final task success.

One implicit trade-off not measured: **action smoothness/consistency across time steps.** The baseline flow matching produces actions via a principled ODE/SDE integration from noise to action. GeCO produces actions via gradient descent optimization initialized from random noise each time step. Whether consecutive action chunks from GeCO are as temporally consistent as those from flow matching (which has the ODE structure enforcing smooth trajectories) is not measured. Temporal consistency of action sequences matters for robot control stability and is a standard CoRL/RSS concern for manipulation policies.

## Framing-vs-Demonstration Check

**Framing (Abstract):** "GeCO improves success rates and efficiency with an optimization-native mechanism for safe deployment"
**Methodology:** Safe deployment is tested only via the OOD detection mechanism, not actual safety metrics (collision rates, joint limit violations, force/torque bounds). "Safety" in the abstract implies robust deployment guarantees; the demonstration shows only anomaly score separability (AUROC).

**Framing (Abstract):** "GeCO learns a stationary velocity field in the action-sequence space where expert behaviors form stable attractors."
**Methodology:** The stationary attractor property is claimed by construction and illustrated in Figure 1, but never verified experimentally with an attractor analysis (Lyapunov stability tests, convergence rate measurements across diverse initial conditions). The paper verifies convergence empirically on the benchmarks but not the claim that expert behaviors specifically form attractors (vs. the optimizer just happening to converge near good actions due to the training signal).

**Framing (Sec. 4.1):** "GeCO with a budget of only 5 steps already outperforms Rectified Flow (20 steps), achieving a higher success rate (91.9% vs. 90.0%) while requiring **75% less compute**."
**Denominator check:** "75% less compute" uses NFE as the compute proxy. 5 NFE / 20 NFE = 75% reduction. This is fair in terms of model evaluations but does not account for per-step computation cost differences (step-size schedule computation, convergence check computation in lines 5-7 of Algorithm 1). The framing "75% less compute" without wall-clock qualification is a soft overstating of the efficiency claim — though not clearly a denominator switch in the strict sense.

**Framing (Sec. 4.2):** "π_0 with GeCO achieves the strongest overall performance with an average success rate of **0.36**, improving upon the baseline π_0 (0.294)."
**Actual numbers:** On LIBERO (Table 4), GeCO (π_0) = 93.9 vs. π_0 baseline = 94.2 — GeCO is actually *lower* for π_0 on LIBERO. The "strongest overall performance" claim in Sec. 4.2 refers to VLABench (Table 3), where GeCO π_0 = 36.0 vs π_0 baseline = 29.4. The narrative in Sec. 4.2 blends VLABench gains with LIBERO comparisons without always making clear when GeCO underperforms on LIBERO for π_0.

## Mechanism-Attribution Audit

GeCO introduces multiple simultaneous novelties:
1. **Time-unconditional training** (γ not provided to model, Sec. 3.2)
2. **Velocity rescaling schedule c(γ)** (Eq. 2, Sec. 3.2 — ensures stable equilibrium at γ→1)
3. **Adaptive gradient-based inference** (Algorithm 1 — iterative optimization instead of fixed-schedule ODE)
4. **OOD detection from field norm** (Sec. 3.4 — emergent property of (1)+(2))

**Ablation coverage:**
- Table 6 ablates the truncation parameter α ∈ {0.6, 0.8, 0.9} on LIBERO — this tests sensitivity of the c(γ) onset parameter, NOT isolated component contributions.
- No ablation isolates time-unconditional training (component 1) from adaptive inference (component 3): the paper does not test "time-unconditional field with fixed-schedule inference" as a standalone baseline. This would answer: does the performance gain come from the stationary field itself, or from the adaptive inference mechanism?
- No ablation tests "standard time-conditional policy with adaptive early stopping": if adaptive inference is the primary driver of efficiency gains, a time-conditional model with convergence-based early stopping might achieve similar NFE savings without requiring the time-unconditional reformulation.
- The coupling of (1)+(2)+(3) is never decoupled in the ablation. The paper attributes gains to the overall GeCO framework, but the mechanism credit (is it the field geometry or the adaptive inference?) is unresolved.

This is a **coupled-novelty mechanism attribution failure** — the key question of whether the time-unconditional field is necessary (versus time-conditional with early stopping) is never answered.

## Soundness Score Rationale

**Primary trigger: Coupled-novelty mechanism attribution failure (Score 2 candidate)**

The paper introduces at minimum two deeply coupled novelties: (A) time-unconditional stationary field (no γ input to model) with velocity rescaling c(γ), and (B) adaptive convergence-based early stopping at inference. The central efficiency claim — "75% less compute while matching/exceeding performance" — is attributed to the joint system. Table 6 ablates only the truncation parameter α (sensitivity analysis), not component isolation. The critical missing baseline is: **time-conditional flow matching with adaptive early stopping** (terminate when gradient norm falls below τ_opt). If this baseline achieves similar NFE savings at similar success rates, then the time-unconditional reformulation (the paper's primary architectural contribution) adds no measurable value over a simple convergence criterion on a standard model.

The paper's framing presents the time-unconditional field as the mechanism enabling adaptive inference: *"the stationary geometry yields an intrinsic, training-free safety signal"* and *"test-time inference becomes an adaptive process"*. But convergence-based early stopping can be applied to any iterative solver, including standard ODE solvers for time-conditional flow matching. The claim that the stationary field is *necessary* for adaptive inference is never empirically tested.

**Secondary issue: Wall-clock compute claim without timing data**

"75% less compute" is stated without any wall-clock measurement. GeCO's inference loop computes f_θ(a^(k), s) repeatedly in a gradient-descent optimization loop; Rectified Flow integrates an ODE. NFE counts are comparable only if per-step costs are equal, which is not shown. This is a softer version of the "cost claim without measurement" score-2 trigger — the paper's NFE-based compute comparison is reasonable as a model-evaluation-count metric, but the gap between "75% fewer forward passes" and "75% less compute (time/FLOPs)" is never bridged.

**Real-world results: Low seed count without variance**

Table 7 reports 10 trials per task (2 tasks). With binomial outcomes and small n=10, confidence intervals are wide and overlapping between baseline (10%, 20%) and GeCO (70%, 80%). No confidence intervals are reported. The statistical significance of the real-world result cannot be assessed.

**Assessment:**
The coupled-novelty attribution failure is real: the key missing baseline (time-conditional + adaptive early stopping) is never run. However, this is a conceptually new method with a clean theoretical motivation, and the LIBERO simulation results (Table 1, 50 rollouts) are large-n comparisons where the performance trends are credible. The construct-validity concerns (NFE vs wall-clock, OOD scope) are legitimate but do not represent data contradictions.

The mechanism-attribution failure is serious for the paper's theoretical framing but the empirical performance claims (GeCO outperforms fixed-schedule baselines in success rate and NFE) are themselves credibly supported by the simulation results. The issue is that the *reason* for the improvement is unestablished.

**Score: 3** — No single score-2 trigger fires with full certainty (the coupled-novelty ablation gap is real but the paper's practical claims are credibly demonstrated; the compute claim is presented in NFE terms which is a defensible proxy). Multiple score-3-tier flaws apply: (1) missing component-isolation ablation for the method's two coupled novelties, (2) wall-clock compute not reported despite "75% less compute" framing, (3) OOD evaluation on narrow suite-level shift without broader generalization testing, (4) real-world results with n=10 and no confidence intervals. No clearly well-executed offsetting section — the ablation is only sensitivity analysis of one hyperparameter; LIBERO results are strong but the absence of the key baseline (time-conditional + early stopping) is the main gap. Score 3 over 4 because the joint novelty attribution is unresolved on the paper's primary mechanism claim.

**Confidence: 4** — read full paper and appendix; rationale anchored to Algorithm 1, Table 1, Table 6, Table 7, Eq. 2-3, Sec. 4.2; judgement is firm.
