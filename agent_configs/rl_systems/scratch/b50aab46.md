# b50aab46 — Draft-Conditioned Constrained Decoding for Structured Generation in LLMs (DCCD)

**Domains:** d/NLP, d/Generative-Models. **Cluster routing:** in scope.

## Load-bearing claim inventory

1. **Abstract.** "DCCD improves strict structured accuracy by up to +24 percentage points over standard constrained decoding (e.g., 15.2% to 39.0% on GSM8K with a 1B model)."
2. **Abstract / Sec 5.2 / Fig. 5.** "Enables smaller model pairs to match or exceed much larger constrained baselines, yielding substantial gains in parameter efficiency."
3. **Sec 4.** KL-projection theoretical view: "constraint-induced distortion is governed by feasible mass; cumulative 'projection tax' under hard constraints."
4. **Abstract.** "Simple two-step, training-free inference procedure that decouples semantic planning from structural enforcement."

## Construct inventory

- **C1 — "improves strict structured accuracy."** Operational metric: joint correctness AND structural validity per Sec 5.1. Multi-dataset (GSM8K, GSM-Symbolic, MATH500, FOLIO), multi-scale (1B–14B). ✓ Comprehensive.
- **C2 — "parameter efficiency."** Operational metric: accuracy / total parameter count (Fig. 5). **Aspect NOT measured: inference cost (FLOPs, wall-clock latency).** DCCD requires running TWO models sequentially. A "1.5B+1.5B = 3B parameter" pair processes the input twice — actual compute is closer to 6B-equivalent FLOPs. The Fig. 5 normalization-by-parameter glosses over the 2× sequential cost. The "parameter efficiency" framing is technically defensible (memory, deployment footprint) but reads more strongly than compute-cost analysis would justify.
- **C3 — KL-projection mechanism.** Operational test: implicit, via the prediction that small models (with lower α(h_t)) benefit more from DCCD. Tab. 2 confirms: 1B model gets +23.8pp on GSM8K, 14B model gets +8.7pp. ✓ The pattern is consistent with the theoretical prediction.
- **C4 — "decouples semantic planning from structural enforcement."** Operational comparison: CD vs DCCD (with K=1 by default). **Missing baseline:** the natural "CoT-then-extract" alternative — generate free-form reasoning first, then constrain only a final-answer-extraction step. This baseline is what production systems often use and is conceptually similar to DCCD's first-stage idea but without DCCD's specific projector-conditioning operationalization. Without this baseline, the paper attributes gains broadly to "decoupling planning from enforcement" when part of the gain may be from the standard CoT-then-format pattern.

## Counter-hypothesis attempts

1. **The improvement may be the standard CoT benefit dressed in projection-tax framing.** DCCD's draft is "free-form planning or intermediate reasoning" (Sec 4.2) — i.e., a CoT generation. A simple "generate CoT freely, then JSON-format the final answer via CD on a short extraction step" would test whether DCCD's specific conditioning structure adds value beyond "do CoT before formatting." Not reported. **Status: unrefuted gap.**

2. **GSM-Symbolic 1B has CD=0.00 baseline (Tab. 2).** The 1B model under CD apparently never produces a strict-valid+correct output on GSM-Symbolic. DCCD pushes this to 9.00 — but CF (Constrained Few-Shot, no decoding-time enforcement) ALSO gets 9.00 on the same cell. So on GSM-Symbolic 1B, DCCD merely matches the easier baseline; "DCCD beats CD by 9pp" is partially because CD fails entirely on this configuration. **Status: directly observable from Tab. 2.**

3. **"Up to +24 percentage points" is the maximum across (model, dataset) configurations.** The median improvement across all 24 cells in Tab. 2 is much smaller. E.g., 14B FOLIO: 18.72 → 25.62 = +6.9; 14B GSM8K: 86.43 → 95.15 = +8.7; 1B FOLIO: 19.70 → 21.67 = +2.0. The "+24" headline is technically honest with the "up to" qualifier but reads more strongly than the median gain. **Status: unrefuted (maximum vs median framing).**

4. **"39% higher confidence" claim minor numerical issue.** Sec 5.3 / Fig. 7: DCCD mean confidence 0.527, CD mean 0.393. (0.527-0.393)/0.393 = 34.1%, not 39%. Minor numerical discrepancy in the claim. **Status: noted; minor.**

## Mechanism-attribution audit

DCCD components:
- (a) Two-stage decoding (free-form + constrained).
- (b) Free-form CoT-style draft generation.
- (c) Best-of-K draft selection (optional, K>1).
- (d) Conditioning the projector model on the draft.

The Tab. 2 / Fig. 4 main results compare DCCD vs CD. The difference is everything: stage 1 + stage 2 + conditioning + (optionally) K. Internal ablations:
- Fig. 6 covers test-time scaling (n candidates + voting) but doesn't separate K=1 from K>1 in the headline.
- No ablation on draft length/type (would test whether ANY auxiliary context helps, vs. specifically a CoT-style plan).
- No CoT-then-extract baseline (would test whether DCCD's specific second-stage conditioning is necessary, vs. simpler post-hoc format extraction).

This is partial mechanism attribution. The "DCCD-vs-CD" macro-comparison shows the joint contribution; the within-DCCD micro-attribution is incomplete.

## Multi-objective trade-off audit

- **Compute cost.** DCCD requires 2 model passes per query. Fig. 5's "accuracy per billion parameters" doesn't reflect this. Wall-clock or FLOPs comparison vs. CD is not reported.
- **Token-generation cost.** DCCD generates a draft (potentially long) + final structured response. Per-query token volume is much higher than CD's single output. Token-budget-based efficiency comparison missing.
- **Memory.** DCCD with two distinct models (e.g., 1.5B + 1.5B) requires loading both. Single-model CD on a 3B model has the same memory footprint, so this is comparable. ✓

## Headline-number denominator audit

- "+24 percentage points": absolute, denominator is CD baseline. Math checks out at 1B GSM8K cell.
- "Parameter efficiency 253% improvement" (Fig. 5): MATH500, 1.5B+1.5B = 3B params at 12.7 acc/B vs. 8B model with CD at 3.6 acc/B. (12.7/3.6 - 1) = 253%. Math correct. Compute-cost adjustment would change this number significantly.

## Silent baseline omission concerns

- **No CoT-then-extract baseline.** "Generate free-form reasoning, then prompt for JSON-formatted final answer (with or without CD)." This is what production systems often do. DCCD's specific innovation over this pattern (formal projector model + KL-projection conditioning) is not isolated.
- **GSM-Symbolic 1B CD=0.00 is suspicious.** A 1B model with constrained decoding apparently NEVER produces a valid+correct output. The headline framing of "DCCD improvement" partially comes from DCCD restoring functionality CD destroyed at small scales. On this cell, DCCD merely matches CF (no decoding-time enforcement at all).
