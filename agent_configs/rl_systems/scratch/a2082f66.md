# Scratch — Molecular Structure-Language Dataset (a2082f66)

## Load-bearing claims

- **Abstract:** "A rigorous validation protocol combining LLM-based and expert human evaluation on a subset of 2,000 molecules demonstrates a high description precision of 98.6%."
- **Abstract:** "The resulting dataset provides a reliable foundation for future molecule-language alignment."
- **Sec. 4.2 Validation Results:** "The validated descriptions achieve an overall precision of 98.6%, with 98.7%, 99.2%, and 98.3% for the easy, medium, and hard categories, respectively."

## Construct inventory

- **Construct: "description precision"** — operationalized as: hybrid validation. (a) LLM validator (GPT-5.2 medium) attempts to reconstruct the SMILES from the description in pass@3 (Sec. 4.2); 1,713/2,000 = 85.7% pass at this stage. (b) Human validators (up to two, three attempts each, exact-match to ground truth) check the 287 LLM-failures; 252+7 ≈ 259/287 = 90.2% pass at this stage. Total: 1,972/2,000 = 98.6%.
- **Construct: "reliable foundation for molecule-language alignment"** (abstract) — operationalized: not directly. No downstream alignment/property-prediction/molecular-design task is evaluated using the dataset. This is forward-looking framing, not demonstrated.

## Counter-hypothesis list

- **CH1 (generator-validator coupling).** The headline 98.6% uses GPT-5.2 as the generator (Table 1: easy=GPT-5.2-high, medium/hard=GPT-5.2-xhigh) AND GPT-5.2 as the LLM validator (Sec. 4.2). Same model family. **The paper's own ablation in Sec. 4.3 (Sensitivity to Generation Model)** decouples this: GPT-5 generation × GPT-5.2 validation gives precision drops of 0.6% (easy), 18.0% (medium), and 20.7% (hard) — i.e., absolute precisions ≈ 98.1%, 81.2%, 77.6%. The paper interprets this drop as "GPT-5 is a worse generator." An alternative interpretation: the GPT-5.2 validator preferentially decodes GPT-5.2-style descriptions, so when generator≠validator-family, precision falls by ~20% on harder samples.
- **Distinguishing CH1 (paper's interpretation: generator capacity) from CH1' (coupling): would require either (i) GPT-5.2 generation × full human-only validation across all 2,000 samples, or (ii) GPT-5 generation × GPT-5 validation, or (iii) GPT-5.2 generation × human-only validation across the 1,713 LLM-passes. None of these are reported. Sec. A.3 notes that fully human-validating 2,000 samples would take ~390 human-hours, which was budget-prohibitive.

- **CH2 (downstream alignment utility).** The dataset's positioned value is for downstream molecule-language alignment training. No such alignment training is performed in the paper; no downstream task (MolLangBench tasks like molecular structure recognition/editing/generation) is evaluated. The "reliable foundation" claim is conjectural.

- **CH3 (atom-matching filter conditioning).** The 98.6% is on the post-filter set (after atom-matching removes ~2.3% of generation candidates). Sec. 4.3: "97.7% pass the atom-matching check"; for atom-match-failures, "precision drops sharply to 27.7%." So the unconditional precision on raw generation output is ≈ 0.977 × 0.986 + 0.023 × 0.277 = 96.9%. Still high but distinct from 98.6%.

## Framing-vs-demonstration check

- "Reliable foundation for molecule-language alignment" → no downstream alignment evaluation. The dataset's stated value proposition is not empirically demonstrated.
- "High description precision 98.6%" → demonstrated, but contingent on (a) post-filter conditioning, (b) generator-validator family coupling. The 98.6% should ideally be reported with these conditioning notes adjacent to the headline.

## Reward / mechanism / cost / circular / silent-baseline checks

- **Circular evaluation tendency:** generator and validator are same model family. Mitigated by human checks of LLM-failures, but unchecked on the 85.7% LLM-passes. Sec. 4.3 ablation provides indirect evidence of coupling (~20% precision swing on medium/hard).
- **Silent baseline omission:** the paper cites Edwards et al. 2021/2024 (Text2Mol, L+M-24) as predecessor molecule-language datasets. No comparison of validation precision with those datasets under the same protocol.
