## Reward / objective inventory

PPO reward (Sec 3.4 Eq. 4): r_t = w_g r_t^goal + w_p r_t^progress + w_c r_t^collision + w_s r_t^speed.
Weights: w_g=400, w_p=5, w_c=-100, w_s=-0.1 (target speed 1 m/s).
Baseline RL policy (Sec 4.1): "PPO agent whose policy shares the same architecture as the value network, except that it directly outputs the action mean and standard deviation. The PPO policy is trained separately for each environment using 10 parallel environments and a total of 1M steps per environment."

**Reward overlap audit.** Table 1 RL Policy: SR=30.59. TIC-VLA-no-RL: SR=47.06. TIC-VLA-full (with PPO): SR=55.29. The +8.23 SR delta from RL is non-trivial but smaller than the +16.47 SR from architecture (TIC-VLA-no-RL vs RL Policy). The headline-mechanism credit goes to "latency-consistent training" but the table actually attributes most of the gain to the IL-stage architecture+training combo.

## Counter-hypothesis list

1. **Baseline-training disparity.** TIC-VLA's gain over NaVILA / DualVLN / Uni-NaVid is mostly due to latency-injection training (Stages 2 & 3 of Sec 3.3) which the baselines didn't get. Fair comparison would re-train baselines with the same delay-injection IL. **Not ruled out**: paper only states "fine-tuned on the same datasets" (Sec 4.1), not on the same training procedure with delays.
2. **Alternative-mechanism: KV-cache features as a generic dense conditioning signal.** Table 2 shows the KV-cache interface beats waypoint interface by 14.6 SR even *without* latency metadata (30.59 vs 16.47). Most of the architectural gain is from "use VLM hidden states as cross-attention K/V" rather than from the "delayed-aware" framing. The paper attributes credit to delayed-semantic-control framing; an equally consistent reading is "hidden-state conditioning > waypoint conditioning." **Not ruled out**: no synchronous-KV-cache baseline that decouples KV-cache from delay-handling.
3. **Coupled novelty: ego-motion offset alone.** Table 9 in appendix shows ego-motion offset alone moves SR from 41.18 to 47.06 (+5.88). So ~36% of the latency-aware-modeling delta is from ego-motion compensation, not from delayed-semantic reasoning per se. **Partially refuted** by the appendix table but the paper main text still credits the "delayed semantic-control interface" as a unitary mechanism.
4. **DynaNav benchmark co-design.** TIC-VLA designers also designed the benchmark and baselines were not co-designed for it. Baselines like NaVILA were designed for different benchmarks; running them on DynaNav without per-task tuning may understate them. **Not ruled out**: paper does not report whether baselines were given per-environment tuning equivalent to TIC-VLA's 3-stage pipeline on DynaNav data.

## Multi-objective trade-off audit

Reward function trades off goal-reaching, progress, collision-avoidance, speed (4 terms). Table 1 reports NE / SR / SPL / CR — so all 4 reward terms map to a measured metric. No obvious unmeasured trade-off axis from the reward design itself. Speed penalty target = 1 m/s; speed itself is not reported in the eval tables, so it's unclear if TIC-VLA actually slows down to game collision avoidance — a generic counter-hypothesis worth flagging but not load-bearing here because SPL (path-length efficiency) is reported and it's high.

But "zero-shot" here means "no real-world training data" — the model was trained on simulation + SCAND + GND (real-world driving datasets). - Table 1 SR comparison: TIC-VLA 55.29 vs NaVILA 28.24. Denominator is the same 85-episode benchmark. No denominator switch.
- Table 8 "Average SR": unweighted mean over 4 task types with disparate n. Each task = 5 trials = 1 SR cell. "Average" is the mean of cell percentages, not the mean over trials. With n=5 per cell each cell's variance is huge.

## Mechanism-attribution audit

The paper introduces three coupled novelties:
1. (A) Delayed semantic-control interface (KV cache features + ego-motion offset + latency metadata). Ablated in Table 2 as a 2x2 of (interface, latency).
2. (B) Latency-consistent training pipeline (3 stages: SFT, IL-with-delayed-inference, RL-with-async-guidance). Partially ablated in Table 1 (sync vs async, no-RL vs full RL) but never with delay-injection training itself disabled while keeping architecture.
3. (C) DynaNav benchmark.

**Critical missing ablation**: a row showing "TIC-VLA architecture, trained with synchronous IL (no delay injection), deployed asynchronously." This would isolate (B) from (A). Without it, "latency-consistent training pipeline" credit is unsupported.

**Critical missing ablation**: a row showing "baseline architecture (e.g., DualVLN or NaVILA) trained with TIC-VLA's latency-injection IL." This would test whether the gain is in the architecture or in the training pipeline.

The Sync row in Table 1 (TIC-VLA Sync. SR=16.31) tells you that if you take the trained TIC-VLA model and force it to run synchronously, performance crashes — but that doesn't tell you whether the training-time delay injection caused the gain.

## Soundness score rationale

The headline claim "consistently outperforms prior VLA models" rests on a comparison where TIC-VLA is the only method given the latency-consistent training pipeline (Sec 3.3 Stages 2 and 3 — IL with delayed inference + RL fine-tuning under async guidance). The baselines (NaVILA, DualVLN, Uni-NaVid) are described as "fine-tuned on the same datasets" (Sec 4.1) — that's data parity, not training-procedure parity. The paper's own Table 1 shows TIC-VLA-Sync (the same architecture without async inference) collapsing to SR=16.31, *worse than* every baseline (NaVILA 28.24, DualVLN 30.59, Uni-NaVid 22.35). This is the natural fingerprint of architecture-coupled-with-training-pipeline gains: the architecture alone is bad, the training pipeline alone (applied to baselines) is untested, and the joint configuration is the only thing that wins.

This is a **coupled-novelty mechanism attribution failure** on a load-bearing claim. The paper credits gains to the delayed semantic-control interface AND the latency-consistent training pipeline, but the ablation table that would isolate them — "baselines trained with TIC-VLA's delay-injection IL" — is missing. A concurrent reading "the gain is mostly from training methodology and could be added to NaVILA / DualVLN" is not refuted by the paper's evidence.

Additionally, the real-world result (Table 3, Table 8) is n=5 per task per method, no CIs, no statistical test, no blinding mentioned. "Consistently outperforms" is a confident phrasing for this sample size.

This fits the score-2 trigger "**Coupled-novelty mechanism attribution failure**" precisely. Score 2.
