# Scratch — CF-HyperGNNExplainer (bd42a4b0)

## Load-bearing claims

- **Abstract:** "Experiments on three benchmark datasets show that CF-HyperGNNExplainer generates valid and concise counterfactuals."
- **Sec. 1 contributions (3):** "We provide an extensive empirical evaluation on three citation-network datasets, **including comparisons with graph-based counterfactual explainers**, an ablation study, and a runtime analysis showing that our method strongly outperforms graph-based baseline methods when adapted to native hypergraph settings."
- **Sec. 8 conclusion:** "Experimental results on three citation network datasets demonstrate that CF-HyperGNNExplainer successfully generates valid and concise counterfactual explanations. Both method variants achieve high accuracy (up to 72.7% on CiteSeer)."

## Construct inventory

- **Construct: "valid and concise counterfactuals on three datasets"** — operationalized via Table 1 (NHP/HP × Cora/CiteSeer/PubMed). Accuracy (validity) ranges 0.506 (PubMed NHP) to 0.727 (CiteSeer NHP). Sparsity is high (0.89-0.999). Explanation size 2.5-3.2 incidences/hyperedges.
- **Construct: "comparisons with graph-based counterfactual explainers across three datasets"** — operationalized via Table 2, which only reports comparisons on Cora. CiteSeer and PubMed have no baseline comparison.
- **Construct: "real-world higher-order interactions"** (Sec. 1 motivation) — operationalized: not directly. The three citation network datasets (Cora, CiteSeer, PubMed) are converted to hypergraphs via 1-hop neighborhood construction (Sec. 5.2: "For each node i in the graph, we define a hyperedge ε_i that contains node i and all its neighbors"). The hypergraphs have N nodes and N hyperedges, each hyperedge being a 1-hop neighborhood. This is a graph-to-hypergraph transformation, not natively higher-order data. No experiments on co-authorship teams, biochemical complexes, group communications, or multi-item user sessions (the Sec. 1 motivating examples).

## Counter-hypothesis list

- **CH1.** The "outperforms graph-based baselines" claim relies entirely on Table 2 (Cora only). On the other two datasets (CiteSeer, PubMed) we have no comparison data. Without comparison numbers, the claim "method strongly outperforms graph-based baseline methods" (Sec. 1 (3)) is anchored to a single dataset.
- **CH2.** PubMed accuracy is 0.506 (NHP) / 0.524 (HP). On nearly half of test nodes the explainer fails to find a valid counterfactual. The paper attributes this to "the dataset's larger scale and sparsity" (Sec. 6.1) without further investigation. Whether CF-GNNExplainer would do similarly poorly on PubMed-as-hypergraph is unknown.
- **CH3.** Random baseline numbers are not empirically reported. Sec. 6.2 includes a theoretical analysis showing random sampling at sparse hypergraphs (d_v=5, t=100) finds the minimal counterfactual with probability >0.95. The text claims random baselines produce "significantly larger explanation sizes" but no random-baseline accuracy/sparsity/explanation-size table is provided. The paper's own theoretical analysis suggests random would be a strong coverage baseline; quality (explanation size) is the appropriate axis for comparison.

## Multi-objective trade-off audit

The method optimizes for prediction-flip + sparsity (L1 distance regularizer, Eq. 7). Reports accuracy, sparsity, explanation size — multi-axis on the right metrics. Adequate.

## Framing-vs-demonstration check

- "Comparisons with graph-based counterfactual explainers" (Sec. 1 (3) framing) → comparisons only on Cora, not CiteSeer/PubMed. Moderate framing-vs-demonstration mismatch.
- "Real-world higher-order systems" (Sec. 1 motivation) → citation networks transformed via 1-hop neighborhoods. Mild framing-vs-demonstration mismatch (citation networks are at least benchmark-adjacent).
- "Up to 72.7% on CiteSeer" (Sec. 8 conclusion) → cherry-picks the highest cell from Table 1; PubMed validity (~50%) not surfaced in conclusion.

## Reward / mechanism / cost / circular / silent-baseline checks

- **Silent baseline omission on a load-bearing comparison:** Tables 1 and 2 are disjoint. The "three datasets" framing in abstract + contributions implies cross-dataset comparison; only Cora has it.
- **Random baseline:** acknowledged in prose, theoretically analyzed, but no empirical numbers reported on Cora/CiteSeer/PubMed.

## Soundness score rationale

Score-3 triggers (multiple):
1. Baseline comparison only on Cora (1/3 datasets) despite "three datasets" / "comparisons with graph-based explainers" framing.
2. Hypergraph data is artificially constructed from graphs (1-hop neighborhoods), not natively higher-order. The "real-world higher-order systems" motivation is not tested on actual higher-order data.
3. PubMed validity ~50% on the largest dataset; the "valid and concise counterfactuals" framing in abstract is mostly true on Cora/CiteSeer, marginal on PubMed.
4. Random baseline numbers not empirically reported despite the paper's theoretical analysis suggesting random is a relevant coverage baseline.

Offsetting strength:
- Two method variants (NHP, HP) at different granularities.
- Hyperparameter sensitivity ablation (Tables 3, 4).
- Runtime analysis (Table 5) showing sparse implementation gives 13.5x speedup.
- Sparse implementation released.
- Method is genuinely the first counterfactual explainer specifically for HGNNs.

→ **Score 3** (multiple unrefuted score-3-tier flaws on load-bearing claims; offsetting strength is real but does not address the baseline-comparison gap or the hypergraph-construction artificiality).

A score-2 case could be argued (e.g., "comparisons with graph-based counterfactual explainers" in contributions list is partially false since comparison is single-dataset). I'm not going there because the paper does include Cora-baseline comparisons honestly and the methodological contribution is real.
