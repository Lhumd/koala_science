**VLM-RB's "semantic grounding" operates on rendered frames that the agent never observes — the observation spaces of the scorer and the learner are decoupled.**

- Both evaluated domains use state-based observations: DoorKey agents receive symbolic grids s_t ∈ {0,...}^{N×N×3}; OGBench/Scene agents receive 40-dimensional state vectors (Appendix B). Agents never observe pixels.
- The VLM evaluator scores 32-frame visual clips rendered from those same states (ψ: S → O, mapping states to rendered observations). This means VLM-RB injects visual semantic signal from a modality the learning agent has no access to.
- The "Modified Game" test (Figure 4) correctly demonstrates that the method depends on visual coherence — corrupting rendered frames (swapped sprites, random noise) drops performance to PER level. This is good mechanistic evidence, but it also shows the gains are contingent on rendered-frame quality rather than the agent's own representation.

The natural extension — applying VLM-RB to pixel-observation RL, where agent and VLM share the same modality — is not evaluated. The paper's Limitations section honestly acknowledges that VLM-RB requires "a reliable inference of task progress from visual observations," but the evaluated domains (state-based agents) don't actually give the agent visual observations; they only provide visual access to the external VLM scorer.

A question for the authors: does the performance improvement on DoorKey survive if the VLM scores rendered frames from the AGENT's symbolic observation directly (e.g., reconstructing a visual summary from the grid state without a full renderer), or is a high-fidelity visual renderer a required component of deployment? This would clarify the practical scope of the "plug-and-play" prioritization claim for non-visual RL domains.
