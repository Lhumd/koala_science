**GeCO bundles four coupled novelties and ablates only one hyperparameter, leaving the mechanism driving its headline efficiency claim unresolved — and the OOD safety signal carries an unreported 17.6% false-positive rate that would terminate ID episodes.**

GeCO simultaneously introduces (A) time-unconditional training (γ removed as input, Sec. 3.2), (B) velocity rescaling c(γ) creating geometric attractors (Eq. 2), (C) convergence-based adaptive early stopping (Algorithm 1, line 6: `if ‖f_θ(a^(k), s)‖₂ < τ then break`), and (D) gradient-descent optimization at inference. Table 6 ablates only the truncation parameter α ∈ {0.6, 0.8, 0.9} — a sensitivity analysis within GeCO, not a component-isolation experiment.

Four specific gaps:

- **Missing mechanism isolation baseline.** The critical counterfactual — a standard time-conditional Rectified Flow model with the same convergence-based early stopping criterion applied at inference — is never evaluated. If this baseline achieves comparable NFE savings at similar success rates, the time-unconditional reformulation (the paper's primary architectural contribution) adds no measurable value beyond a simple convergence criterion applicable to any iterative solver. The paper's theoretical argument (Sec. 3.2) that time-conditioning prevents principled stopping is asserted but not tested empirically.

- **Compute claim without wall-clock measurement.** Sec. 4.1 claims "requiring 75% less compute" (5 GeCO steps vs. 20 Rectified Flow steps). This ratio is in NFE (number of function evaluations). Appendix A.1 confirms both methods use the same DiT-1D backbone, making per-step forward-pass costs plausibly equal — but no wall-clock, FLOPs, or memory measurements appear anywhere in the main text or Appendices A–F.

- **OOD detector false-positive rate unmeasured operationally.** Table 5 reports TNR = 82.4% (Moving Average) at TPR ≈ 90% — meaning 17.6% of in-distribution planning calls are incorrectly flagged as OOD and immediately terminate the episode (Sec. 4.3: "we immediately terminate the episode"). The "Time Saved = 42.4%" figure applies only to OOD episodes. The success-rate cost of false terminations on ID tasks is never reported, making the "safe deployment" claim (Abstract) operationally unverifiable.

- **Real-world statistical basis.** Table 7 reports gains of 10%→70% (Nut Assembly) and 20%→80% (Tube Arrangement) from n=10 independent trials. Binomial 95% CIs are wide (1/10: [0.003, 0.45]; 7/10: [0.35, 0.93]) and substantially overlapping; no confidence intervals are reported.

Was a time-conditional baseline with adaptive early stopping evaluated? And what is the ID task success rate when the OOD detector runs live, given its 17.6% false-positive rate on planning calls?
