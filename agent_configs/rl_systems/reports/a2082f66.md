The headline "98.6% description precision" (abstract; Sec. 4.2) is the load-bearing empirical result. Reading Sec. 4.2 alongside Sec. 4.3 raises one specific question about the validation protocol that I'd value the authors' view on.

The validation pipeline is hybrid: GPT-5.2 (medium reasoning) attempts SMILES reconstruction in pass@3, accepting 1,713/2,000 = 85.7% of samples; human validators then check the 287 LLM-failures. Generator and LLM validator are the same model family (Table 1: descriptions generated by GPT-5.2-high/xhigh; Sec. 4.2: validator is GPT-5.2 medium). The 1,713 LLM-passes (≈86% of the validation set, dominating the headline number) are not independently human-checked.

Sec. 4.3 (Sensitivity to Generation Model) reports a natural decoupling: when descriptions are generated by GPT-5 instead of GPT-5.2 and validated by GPT-5.2, precision drops by 0.6% (easy), 18.0% (medium), and 20.7% (hard) — absolute precisions on medium/hard fall to ~81% / 78%. The paper reads this as "GPT-5 is a worse generator." An alternative reading is generator-validator family coupling: the GPT-5.2 validator preferentially decodes descriptions written in the way GPT-5.2 writes them.

Distinguishing these would require one of: (a) GPT-5.2 generation × full human-only validation on all 2,000 samples (Sec. A.3 notes ≈390 human-hours, budget-prohibitive); (b) GPT-5 generation × GPT-5 validation; or (c) GPT-5.2 generation × human-only validation on a random subsample of the 1,713 LLM-passes (e.g., 100 samples ≈ 20 human-hours). Option (c) seems tractable and would calibrate how the headline 98.6% changes when the validator family is decoupled from the generator.

A second smaller note: the abstract positions the dataset as "a reliable foundation for future molecule-language alignment," but no downstream alignment evaluation (e.g., LLM fine-tuning + MolLangBench tasks) is performed. Demonstrating the alignment-utility claim is naturally future work, but the abstract framing reads as if it were demonstrated rather than conjectural.
