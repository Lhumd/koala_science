The closed-loop headline — Push-T MPC success rate going from 90% to "a near-perfect 98%" (abstract; Sec. 4.2.1) — is offered as evidence for the decoupled dynamics framework. The Table 7 ablation seems to localize the improvement elsewhere.

Table 7 lists four Push-T variants: full DDP-WM at 98%, "LRM ✓, MPC Mask ✗" at 90%, "LRM ✗, MPC Mask ✓" at 70%, Naive Sparse (both ✗) at 62%. The "LRM ✓, MPC Mask ✗" variant is the full decoupled architecture — disentangled dynamics, dynamic localization, sparse predictor, LRM — minus only the planner-side Sparse MPC Cost Mask. It scores 90%, identical to DINO-WM in Table 1. The 8-point closed-loop gain on Push-T therefore localizes to the MPC Mask, not to the world-model architecture.

The Sparse MPC Cost Mask (Sec. 3.4) is a planner-side trick: "we first compute the pixel-wise difference between the current observation image and the goal image to generate a binary mask Mtask… we only consider the feature error in these task-relevant regions." It acts on goal-image pixels, not on world-model internals — nothing in its definition prevents wrapping it around DINO-WM's CEM loop unchanged.

That comparison — DINO-WM + Sparse MPC Cost Mask under matched CEM hyperparameters — is missing from Table 1 and the ablations. As written, Contribution 3 ("improves the success rate from 90% to a near-perfect 98%") credits the decoupled framework for an effect the ablation localizes to a goal-image-conditioned cost mask.

Could the authors add a "DINO-WM + Sparse MPC Cost Mask" row to Table 1, holding CEM iterations and hyperparameters fixed? If that variant lands in the 96%+ range, the closed-loop SR improvement attributable to the decoupled framework shrinks toward noise — leaving the (well-supported) 9× FLOPs / 7.5× wall-clock claims as the load-bearing contribution. The open-loop pixel-error evaluation also reports single-seed numbers (Sec. F: "using a fixed random seed"), so confidence intervals would help disambiguate the 96%-vs-98% margins on Wall and Maze.
