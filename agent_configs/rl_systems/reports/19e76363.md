**The paper's headline "8× sampling efficiency" claim is stated as a deployment cost reduction but is never supported by wall-clock, FLOPs, or per-verification token-count measurements.**

The abstract (lines 35–37) and Section 4.3 (lines 373–378) both identify an "8× gain in sampling efficiency compared to prior reward-based approaches" as a central contribution, explicitly framing it as practical cost savings: "equivalent performance at one-eighth the generator inference cost in practical deployment settings." The efficiency ratio is derived from sampling count alone — Med-TIV at N=4 vs. Med-PRM at N=32 on Figure 3's MedQA curve.

Three anchored observations undercut this framing:

- **The verifier's own inference cost is unaccounted.** Med-TIV's verifier is a multi-turn agent that iteratively formulates search queries, retrieves top-3 documents from a 24-million-snippet FAISS index, and reasons over retrieved evidence before producing a judgment (Sec. 2.2, Table 5 prompt template). The paper reports neither the average number of retrieval steps per verification call, nor token counts, nor wall-clock time per call. Across main text and Appendices A–D, I did not find any timing measurement.

- **The cost-linearity assumption is stated, not demonstrated.** Section 4.3 (lines 375–376) asserts "since inference cost scales approximately linearly with the number of sampled traces," then concludes the 8× ratio translates to 8× cost reduction. This is only valid if Med-TIV's per-trace verifier cost equals Med-PRM's. Med-PRM uses static RAG (single pre-fetch pass), while Med-TIV uses iterative dynamic retrieval — making equal per-trace cost an extraordinary assumption that requires empirical support.

- **Table 3 shows tool integration adds only +0.94 pp** (Qwen2.5-7B: RL alone 69.60% → RL+Tool 70.54% on MedQA), while RL adds +8.64 pp — the retrieval mechanism central to the efficiency narrative is also the minor contributor. A second framing issue: the Introduction (line 103) states Med-TIV "surpasses the performance of models up to 4× larger in scale," but Table 1 shows Med-TIV = 75.26% MedQA vs. Qwen2.5-32B = 75.26% — an exact tie, not a surpass.

Was the 8× figure validated by any end-to-end wall-clock or total-token comparison? Without such a measurement, the efficiency claim remains a generator-sampling-count ratio that does not account for the multi-turn iterative verifier's substantial per-trace overhead.
