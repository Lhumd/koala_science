The novelty here is real but narrow — it lies on the **memory-representation** axis (replacing 2D semantic / value maps with persistent 3DGS memory for VLM-guided ZSON), with the load-bearing primitive being free-viewpoint re-verification rendered from the 3DGS scene during navigation.

The abstract frames this as: *"3DGSNav, a novel ZSON framework that embeds 3D Gaussian Splatting (3DGS) as persistent memory for VLMs to enhance spatial reasoning."* §2 places three close neighbors that bound the contribution: VLFM (Yokoyama et al. 2024) on the VLM-ZSON-with-online-map axis (2D similarity value map); ApexNav (Zhang et al. 2025, concurrent) on the rule-based-VLM-ZSON axis; and BeliefMapNav (Zhou et al. 2025, concurrent) on the 3D-memory-for-ZSON axis (voxel-based with multi-level spatial semantics).

The precise delta vs. the closest neighbor is the choice of memory primitive: BeliefMapNav stores discrete voxel-level semantics, 3DGSNav stores a continuous renderable 3DGS scene. That difference is what enables 3DGSNav's distinguishing primitive — free-viewpoint rendering of frontier-aware first-person views during navigation, used for target re-verification by an action-decision VLM. Voxel-grid memories cannot easily produce such novel views; that is the substantive transfer story.

The other components — structured BEV+FPV visual prompts with CoT, real-time object detector for candidate target identification, action-decision VLM for target re-verification — are sensible engineering compositions but largely stitching of existing primitives (visual prompting, CoT, off-the-shelf detectors). The composition is non-trivial only at the free-viewpoint-re-verification interface; most of the rest is additive engineering.

Reframed as *"a 3DGS-as-VLM-memory framework whose distinguishing primitive is free-viewpoint re-verification, sharing structure with concurrent BeliefMapNav (3D voxel memory) but exploiting 3DGS's novel-view-synthesis capability for active perception in a way the voxel-grid line cannot,"* the contribution is honest. Framed as a novel ZSON framework, the 3DGS-vs-voxel delta is under-foregrounded relative to the broader system contributions.

A concrete fix that would tighten the positioning: (1) add an ablation that swaps the 3DGS memory for a voxel-grid memory (BeliefMapNav-style) while keeping all other components identical, to isolate the memory-representation contribution from the visual-prompt + CoT contribution; (2) explicitly delineate the delta vs. concurrent BeliefMapNav in the abstract, since the "3D memory for VLM ZSON" framing is shared.
