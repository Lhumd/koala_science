# Novelty Audit — LLM Rubric Representations (3a80b7b7)

**The novel axis is the "global rubric" concept: an LLM synthesizes a programmatic specification from a small diverse subset of examples that standardizes heterogeneous inputs for downstream supervised learning models, enabling sample-efficient learning without domain-specific feature engineering.**

## Decomposition

- **LLM for representation/embedding (prior art):** Using LLMs to produce embeddings or features for downstream tasks is well-established. The rubric approach differs in that it produces an explicit, interpretable programmatic specification — not opaque embeddings — that can be inspected and applied consistently to all inputs.
- **Global rubric (the novel piece):** The LLM analyzes a diverse subset of examples in-context to synthesize a "global rubric": a structured spec that extracts and organizes evidence from naive text serializations into a standardized format. This is more specific than "use LLM for feature engineering" and produces a reusable artifact per dataset.
- **Local rubrics (incremental):** Task-conditioned summaries generated on-the-fly; this is closer to standard prompt engineering and less distinctive from prior work.

## Reframe

The contribution is most accurately framed as: "an agentic pipeline where an LLM synthesizes a dataset-level specification (global rubric) that standardizes heterogeneous input representations for downstream supervised learning, reducing dependency on domain-specific feature engineering." The "streamline" claim in the title is slightly overclaiming on efficiency (LLM calls add latency).

## Concrete fix

A direct comparison against AutoML/feature engineering baselines (e.g., featuretools, TabPFN) on the medical and financial datasets would clarify the specific advantage over existing representation automation. Currently baselines are mostly naive text serialization, which is a weak comparison.
