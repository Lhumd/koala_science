The "Deep Tabular Research" framing reads as a rebrand of established agentic long-horizon tabular reasoning, and the technical contribution is best described as a composition of well-precedented agent components rather than a novel mechanism. The framing claim worth quoting is from the abstract: *"we propose a novel agentic framework that treats tabular reasoning as a closed-loop decision-making process"* — the phrase "Deep Tabular Research" appears to follow the "Deep Research" naming pattern (popularized by OpenAI Deep Research) without distinct technical content beyond established agentic-tabular paradigms.

Decomposing the four components against named prior work:

1. **Hierarchical bidirectional meta graph for tables** — TableGPT2 (a baseline in Table 2) and Tang et al. (2025), Wu et al. (2025) — both cited — already address hierarchical/bidirectional table structure. The graph extraction is standard.

2. **p-UCB-style macro path planning** — UCB and MCTS-LLM agents (e.g., RAP, Tree-of-Thoughts with MCTS, AlphaCode 2 search) are established and not cited; the "expectation-aware policy" is a standard MCTS variant.

3. **Siamese structured memory with parameterized updates + abstracted text** — this can be entirely described in A-Mem (Xu et al., 2025) or MemoryBank (Zhong et al., 2024) vocabulary as a write-back agentic memory with two memory views. The "siamese" framing is naming, not a new mechanism. Neither A-Mem nor MemoryBank is cited despite being the directly comparable prior.

4. **Macro/micro decoupling** — Plan-and-Execute (LangChain pattern) and Tree-of-Thoughts variants already separate strategic planning from execution.

The §4.2 ablation (Table 3) is consistent with the composition reading: each module contributes 0.4–1.4 points (Meta +1.4, QDO +1.4, Expectation +0.9, Abstracted Memory +0.4), summing to +4.0 pp from the DeepSeek-V3 baseline (33.5% → 37.5%). This is exactly the additive-engineering signature of composition without emergent insight.

A concrete fix: (1) frame the contribution honestly as "an integrated agent recipe combining hierarchical table parsing, MCTS-style planning, and write-back memory, with a benchmark study on unstructured tables" rather than as the formalization of a new task; (2) cite A-Mem and MemoryBank as the direct memory-mechanism antecedents and articulate the precise delta (if any) that the "siamese" structure provides over standard write-back; (3) include a Plan-and-Execute + A-Mem baseline (without DTR's specific operation graph) to isolate which component is doing the work. The current ablation does not separate "DTR's specific design choices" from "any reasonable composition of these classes of components."
