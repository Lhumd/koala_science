# 7920483a — Compression as Adaptation: Implicit Visual Representation with Diffusion Foundation Models

## Verbatim abstract
"Modern visual generative models acquire rich visual knowledge through large-scale training, yet existing visual representations (such as pixels, latents, or tokens) remain external to the model and cannot directly exploit this knowledge for compact storage or reuse. In this work, we introduce a new visual representation framework that encodes a signal as a function, which is parametrized by low-rank adaptations attached to a frozen visual generative model. Such implicit representations of visual signals, e.g., an 81-frame video, can further be hashed into a single compact vector, achieving strong perceptual video compression at extremely low bitrates."

## Methodology setup (problem definition only)
Encode a visual signal (image, video) as a LoRA adapter on a frozen pretrained diffusion video model. The LoRA is overfit per-signal so that decoding = run frozen generator with the LoRA. Then map the LoRA params to a single compact vector (citing Li et al., 2025 — likely a hash/quantization scheme). Inference-time scaling at decode for additional fidelity.

## 4b. Closest-prior-work identification

- **Neighbor 1: COIN / COIN++ (Dupont et al., 2021/2022) — cited extensively** — proposed "compression with implicit neural representations" via per-signal MLP overfitting. MemCoder's delta: replaces the standalone MLP with LoRA on a *pretrained generative* model, leveraging the prior. The conceptual move (overfit a function as your bitstream) is COIN's; the use of a strong generative prior is the new axis.
- **Neighbor 2: Uni-LoRA (Li et al., 2025) — cited verbatim in §3.2** — "a closely related formulation has recently been proposed under the name Uni-LoRA" — the one-vector hashing trick is borrowed directly. The paper acknowledges concurrent/prior work explicitly.
- **Neighbor 3: Diff-C (Theis et al., 2022) — cited extensively** — relative entropy coding with diffusion as a prior; the inference-time importance-sampling algorithm in §3.3 is explicitly identified as a "variant of the Diff-C algorithm." The paper writes: *"this procedure can also be viewed as a variant of the Diff-C algorithm (Theis et al., 2022)."* The encoder-side compute scaling is a natural extension within Diff-C's framework.
- **Neighbor 4: "Lossy Image Compression with Pretrained Diffusion Models" (Yang & Mandt, 2023; e.g., DPS-based codec, RCDM) — the line of work using diffusion FMs as compression priors** — these papers use diffusion as a posterior-sampling decoder; this paper uses it as a representational substrate. The delta is that VOV's bitstream IS the LoRA, not a latent passed through diffusion as a denoising decoder.
- **Neighbor 5: HiNeRV / NeRV / E-NeRV (Chen et al., 2021/2023, cited)** — INR-based video compression. Same conceptual move as COIN but for video. Delta as in Neighbor 1.
