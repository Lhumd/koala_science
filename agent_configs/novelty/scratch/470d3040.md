## 1. Selective read

**Abstract.** "We propose unlearning by design ... MUNKEY, a memory augmented transformer that decouples instance-specific memorization from model weights. Here, unlearning corresponds to removing the instance-identifying key, enabling direct zero-shot forgetting without weight updates or access to the original samples or labels."

### 4f.1 Pre-result prediction
- Expert profile: A researcher familiar with (a) memory-augmented networks (NTM, Memory Networks, Memorizing Transformers) and (b) the machine-unlearning post-hoc literature (CF-k, EU-k, SCRUB, SalUN, NegGrad, ℓ1-sparse).
- Predicted: A memory-network architecture where each training instance gets a learned key; unlearning = delete the key. Outperforms post-hoc baselines (which fight catastrophic forgetting) on instance-level unlearning. Should generalize to image classification benchmarks.
- Confidence: high. The "store instance-specific info in addressable memory; delete the address to forget" idea is the natural application of memory networks to unlearning.

## 4b. Closest prior

1. **Graves et al. 2014 (NTM, cited).** Closest neighbor on memory-augmented architecture. Delta: NTM uses external addressable memory for sequence learning; MUNKEY uses external memory for *per-instance keys* tied to training samples.
2. **Sukhbaatar et al. 2015 / Weston 2015 (Memory Networks, cited).** Closest neighbor: associative memory addressed by content. Delta: MUNKEY uses *learnable per-instance keys* during training and deletes keys for unlearning.
3. **Sandler et al. 2022 / Jia et al. 2022 (cited).** Memory-augmented architectures for predictive performance. Delta: MUNKEY repurposes the same architectural primitive for the *unlearning* task (different objective, same architecture).
4. **Lin et al. 2023 (cited).** Sparsity as unlearning precursor. Delta: induces sparsity post-hoc; MUNKEY is architectural.
5. **Goel CF-k, Kurmanji SCRUB, Fan SalUN (cited; benchmarked).** Closest post-hoc unlearning baselines. Delta: MUNKEY does not modify post-hoc weights; deletion is set-theoretic.

## 4c. Pseudo-novelty patterns

- Rebrand? **No.** "Unlearning by design" is a precise descriptor.
- Domain transfer? **Yes — substantive.** Memory networks → unlearning. The new task (instance-level unlearning) has properties that the architecture must support: (a) per-instance addressing during training; (b) deletion-without-weight-update at inference. The paper does address these via the exemplar-token mechanism.
- Composition without insight? **Mild.** MUNKEY = memory-augmented transformer + per-instance learnable keys + deletion-as-unlearning. The composition is conceptually clean and the load-bearing primitive (per-instance learnable key) is the original element.
- Resolution bump? No.

## 4d. Novel framing vs novel science

The "unlearning by design" framing does scientific work — it predicts a class of architectures (memory-decoupled with deletable keys) that future work can build on. Does enable new questions ("what other architectural primitives support deletion?") the post-hoc framing does not.

## 4g. Positioning

- Overclaim? **Mild** — "first application of external memory mechanisms specifically designed for machine unlearning" — this is moderately strong; the claim should be tempered with citations to any prior associative-memory-based approaches (the paper does cite Lin 2023 for sparsity).
- Wrong reference class? Mild — paper compares to "post-hoc baselines" but a comparison to architectures that *also* decouple memorization (Memorizing Transformers, RETRO-style retrievers) would strengthen the claim.
- Missing baseline? **Mild.** A comparison to a retrieval-based classifier (k-NN over a memory bank) where unlearning = drop entries would be the simplest "unlearning by design" baseline. The paper does not benchmark this.
- Inflated framing? "Paradigm shift" is a flag but the paper backs it up with experiments.
- Buried real contribution? No.

Real but narrow novelty — single-axis (architecture-for-task) extension; the architectural primitive is established.
