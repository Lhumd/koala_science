# 0a07cb4f — V_1: Unifying Generation and Self-Verification for Parallel Reasoners

## Verbatim abstract
"Test-time scaling for complex reasoning tasks shows that leveraging inference-time compute, by methods such as independently sampling and aggregating multiple solutions, results in significantly better task outcomes. ... While existing approaches typically evaluate candidates independently via scalar scoring, we demonstrate that models are substantially stronger at pairwise self-verification. Leveraging this insight, we introduce V1, a framework that unifies generation and verification through efficient pairwise ranking. V1 comprises two components: V1-Infer, an uncertainty-guided algorithm using a tournament-based ranking that dynamically allocates self-verification compute to candidate pairs whose relative correctness is most uncertain; and V1-PairRL, an RL framework that jointly trains a single model as both generator and pairwise self-verifier."

## Methodology setup
Two contributions: (1) **V1-Infer**: at test time, instead of scoring N candidates independently with scalar scores, do a Swiss-system tournament with pairwise comparisons, allocating compute to most-uncertain pairs. (2) **V1-PairRL**: co-train a single model as generator + pairwise self-verifier with an online RL objective so the verifier stays in-distribution as the generator improves. Pairwise comparisons are claimed to be more reliable than independent scalar self-scoring.

## 4b. Closest-prior-work identification

- **Neighbor 1: Pairwise LLM-as-judge literature (Zheng et al. 2023 "Judging LLM-as-a-Judge with MT-Bench and Chatbot Arena", Bradley-Terry RLHF literature, refs [10, 43] in paper)** — the observation that pairwise comparison is more reliable than pointwise scoring is established here. Delta: V1 applies this to *self*-verification at test-time scaling and during co-training, rather than to external judging or reward modeling.
- **Neighbor 2: AggLM (Zhao et al., ref [56]) — cited verbatim** — RL-trained aggregator for parallel reasoning that synthesizes a final answer for majority-voting-style aggregation. Delta: AggLM uses a separate aggregator and pointwise reward; V1-PairRL uses a single co-trained model with pairwise reward.
- **Neighbor 3: Generative verifier co-training (refs [18], [33], [46]) — cited as the closest co-training neighbors** — these approaches co-train generator + verifier but use pointwise verification rewards. V1-PairRL's delta is the pairwise reward + online co-evolving objective. The paper explicitly contrasts: "they predominantly rely on pointwise verification rewards. Our V1-PairRL framework advances this by co-training a single model for pairwise self-verification."
- **Neighbor 4: RSA (recursive self-aggregation, ref [44]) — cited concurrent** — alternative to pairwise verification that aggregates instead of selects. V1's contribution is showing RSA's diversity-collapse failure mode and that pairwise self-verification is orthogonal (preserves Pass@N).
- **Neighbor 5: Process Reward Models / step-level verification (Lightman et al. 2023, OpenAI / DeepMind 2023-2024)** — alternative verification approach using fine-grained step-level scoring. Delta: V1 is candidate-level pairwise verification, not step-level scoring; complements rather than replaces PRMs.
