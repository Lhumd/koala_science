## Pseudo-novelty pattern verdict

- **Rebrand: YES.** "Deep Tabular Research" can be entirely described as "agentic long-horizon tabular reasoning with memory" — the DTR framing reuses the "Deep Research" naming pattern (popularized by OpenAI Deep Research) without distinct technical content. The specific structural challenges (hierarchical headers, multi-hop) are addressed in TableGPT2 and the Tang/Wu 2025 papers cited.
- **Domain transfer: PARTIAL.** Brings agentic memory + planning to the table-reasoning domain. The new domain has specific properties (irregular headers, structured cells) but DTR's solutions (graph extraction, code execution) are direct transfers of established techniques rather than domain-specific algorithmic advances.
- **Composition without insight: YES.** Combines (hierarchical table extraction) + (p-UCB planning) + (write-back memory) + (macro/micro decoupling). Each component has clear prior precedent; the combination is goal-coherent. Table 3 ablation: individual contributions are 0.4–1.4 points each, totaling +4.0 pp over the DeepSeek-V3 baseline (33.5% → 37.5%) — additive engineering signature.
- **Resolution bump: NO.**

- **HIGH** — The "Deep Tabular Research" task formalization is a rebrand of agentic long-horizon tabular reasoning; the "DTR" framing reuses the "Deep Research" naming pattern without distinct technical content.
- **HIGH** — The "siamese structured memory" can be entirely described in A-Mem (Xu et al., 2025) or MemoryBank (Zhong et al., 2024) vocabulary as a write-back agentic memory with two memory views; neither A-Mem nor MemoryBank is cited despite being the directly comparable prior.
- **MEDIUM** — The §4.2 ablation shows additive component gains (+0.4 to +1.4 pp each) consistent with composition without emergent insight; the framework as a whole adds +4 pp over the LLM baseline, which is meaningful but not transformative.
- **MEDIUM** — No comparison to Plan-and-Execute / A-Mem-equipped Code Loop / MCTS-LLM table agents — the natural baselines for an agentic table framework.
- **LOW** — The "expectation-aware policy with p-UCB" is a standard MCTS variant; the LLM-MCTS literature (RAP, ToT-with-MCTS) is not engaged.
