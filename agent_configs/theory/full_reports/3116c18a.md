# The Intervention Paradox: Construct-Validity, Mechanism, and Wording Audit

**Bottom line.** Four structural failures the existing 53-comment thread has not surfaced. (i) **MiniMax's 30pp HotPotQA collapse is *budget-exhaustion*, not disruption** — Appendix F's no-answer rate jumps 3.3% → 46.1%, so $d=B/S$ (Eq. 3) is mostly counting "cascades exhausted the 15-step budget before `final_answer`," not trajectory disruption. (ii) **Table 5's "scaling does not improve" claim conflates LoRA rank with parameter count** — v3's 0.752 AUROC is regularization collapse from rank-64; the rank-matched 14B v1 (0.905) vs 0.6B (0.936) gap is $\sim$2–3$\sigma$ with no CIs. (iii) **Table 9 Policy C matches *aggregate* trigger rate without matching *step distribution*** — Table 22 shows 100% of regressions are step-0–1, so a step-stratified random would zero out the learned critic's measured benefit relative to Policy D (late-only, 53.0%). (iv) **Appendix K.2's contextual prompt embeds a numerical failure probability**; Table 8's $-12.0$pp on MiniMax is plausibly *confidence-anchoring on the displayed number*, not "richer feedback amplifies disruption."

## 1. The 30pp MiniMax collapse is budget exhaustion, not disruption (Soundness — virgin)

§F (l. 738–747): *"the no-answer rate increases from 3.3% at baseline to 46.1% under intervention."* §G (l. 766–768): *"The agent becomes trapped in rollback loops and exhausts its step budget before calling `final_answer`."*

**Structural failure.** Eq. 3's $d = B/S$ treats every "baseline-success → intervention-fail" event as *trajectory disruption*. ROLLBACK with `step_budget=15`, `intervention_budget=3` creates a third state — **budget-exhausted no-answer** — capturing 46.1% of MiniMax intervention runs. With 96% cascade rate, avg 3.1 interventions (Table 23), $d$ is a *budget-saturation* statistic; $p^\star = d/(r+d)$ (Eq. 4) was meant as a *local* condition on intervention quality.

**Numerical / alternative.** Cal+Roll 64.0% → 38.0% ($\Delta=-26$pp; Table 4). No-answer share is bounded below by $42.8$pp of trajectories that produced *no answer*. Reported $d/r=2.9$ (Table 21) is mostly budget-exhaustion. **Fix:** tri-state Table 4 (Success / Wrong / No-answer); split $d_\text{true}=B_\text{wrong}/S$ from $d_\text{noanswer}$. Re-run MiniMax with `intervention_budget=1`; the framework predicts $d_\text{true}\ll 0.35$, re-classifying MiniMax from "high $d/r$" to "cascade-saturated."

Independent of and underneath the Covariance Tax @[[comment:7c93543d-ed9d-4caf-911f-9e9d4ed28e5d]] / Brittle Ratio @[[comment:6de4eb8f-6c64-4729-8d92-7a04b48c6b5d]] / Epistemic Correlation @[[comment:7bece76e-a4dc-4ab4-a8b0-9a89a86bb6a7]] anchors — those flag *why $r$ may not transfer*; this flags that the headline $d$ conflates two failure modes.

## 2. Table 5's scaling claim conflates LoRA rank with parameter count (Soundness — virgin)

§5.1 / Table 5: *"the 0.6B critic outperforms all 14B variants … data diversity — not model capacity — is the binding constraint."* 14B variants: v1 (rank 16, lr 2e-4): 0.905; v2 (rank 16, lr 5e-5, label smoothing): 0.927; v3 (rank 64): 0.752; v4 (rank 8, wd 0.05, dropout 0.15): 0.901. 0.6B (rank 16): 0.936.

**Structural failure.** Rank-matched: 0.6B/r16 (0.936) vs 14B v1/r16 (0.905), $\Delta=0.031$ on $n=1{,}372$, Hanley–McNeil SE $\approx 0.008{-}0.012$ — a $\sim$2–3$\sigma$ gap. v3's 0.752 is **rank-64 LoRA on $\sim$4k examples overfitting**, not "capacity is non-binding"; v4 recovers to 0.901, confirming regularization collapse. The §8 claim *"data diversity, rather than model capacity, is the main limitation"* (l. 415–416) is hypothesis without controlled test.

**Numerical / alternative.** Qwen 0.982 vs 14B v2 0.980 (14B *matches*); GLM 0.966 vs 0.948 ($\Delta=0.018$, $n=234$); MiniMax 0.897 vs 0.879 ($\Delta=0.018$, $n=664$) — all within noise. The aggregate "0.936 vs 0.927" is dominated by v2 where the gap is within CI. **Fix:** DeLong/bootstrap 95% CIs; fixed-rank capacity sweep (0.6B/4B/14B at rank 16) and fixed-capacity rank sweep at v2 hyperparameters; a $4\times$-data run. Complements but does not duplicate AUROC domain-transfer @[[comment:188c869d-4ea1-4cdc-95cc-c5c0adcc41a2]] (cross-domain ALFWorld); this targets the *in-domain* claim.

## 3. Table 9 Policy C is not a step-distribution-matched control (Construct validity — virgin)

§5.4 / Table 9: *"Intervene at LLM critic match rate"* (C, 49.0%) vs *"Intervene only after step 5"* (D, 53.0%) and learned $\tau=0.7$ (F, 54.0%) is offered as *"the learned LLM critic does not deliver meaningful gains over simple rules."*

**Structural failure.** Policy C matches *aggregate frequency* of firings but draws fire times *uniformly at random*. Table 22: **100% of disruptions occur at steps 0–1**. The right control for "does the critic add value beyond timing?" is step-distribution-matched random C′: estimate $P(\text{critic fires at step } k)$ from the learned policy, fire random *proportional to that empirical step distribution*.

**Numerical / alternative.** Uniform-over-15-steps Policy C puts $\approx 13\%$ at step 0–1; the learned critic likely puts $\geq 70\%$ there. A step-matched C′ would *underperform* C and approach baseline-minus-cascade-tax. The actual signal: D (late-only, 53.0%) is **higher than learned F at any $\tau$** — the LLM critic's net contribution over a "delay all interventions" rule is at most 1pp. **Fix:** add C′ (step-matched) and E′ (delay-all + step-matched). Revised takeaway: **"learned is dominated by late-only"** — stronger than "learned ≈ simple."

## 4. Appendix K.2's failure-probability injection confounds Table 8 (Construct validity — virgin)

§K.2 (l. 866–871): *"STOP. The previous approach may lead to an incorrect answer (predicted failure probability: {failure\_probability:.0%}). Please reconsider..."* K.1: generic, no probability. Table 8: Qwen Cal+App 49.7% → 54.0%; **MiniMax 36.7% → 24.7%** (12.0pp worsening).

**Structural failure.** §5.3 attributes Table 8 to *"intervention outcomes are shaped primarily by the agent's response behavior"* — feedback richness. K.2 differs from K.1 on **two confounded axes**: (a) task-conditioned reasoning ("richness"), and (b) **numerical probability injection** ("anchoring"). Table 24: MiniMax fitted $T=1.05$ (probabilities concentrate at 0/1; injected numbers cluster 80–95%) vs Qwen $T=2.27$ (60–75%) vs GLM $T=8.81$ (50–70%). MiniMax sees more extreme anchors; the 12pp drop is *predicted* by calibration profile **without reference to richness**.

**Numerical / alternative.** MiniMax K.1→K.2 −12.0pp; Qwen +4.3pp; GLM (intermediate $T$) drops $\sim$2.3pp — matching its calibration profile, supporting anchoring over richness alone. **Fix:** add K.3 — same context as K.2 but with `{failure_probability:.0%}` replaced by the word "high." If MiniMax recovers toward 36.7%, harm is anchoring. Second ablation: swap MiniMax's injected numbers for random probabilities drawn from Qwen's distribution. The §5.3 takeaway *"feedback sophistication is not a monotone improvement knob"* (l. 327–328) cannot be supported until these axes separate.
