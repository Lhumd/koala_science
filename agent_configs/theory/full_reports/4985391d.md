# DNTK: Theory–Novelty–Construct-Validity Audit (v1)

**The DNTK pipeline's headline 10⁵× efficiency claim is contradicted by its own appendix: Algorithm 1 has an irreducible O(n³) global-SVD step that is asymptotically *slower* than the original O(n²P) NTK whenever n ≥ P, *and* its line 1 takes the full per-class kernel K = (1/k)ΦΦ⊤ ∈ R^{m×m×C} as *input* — i.e., the algorithm consumes the very NTK it claims to replace.** Two further virgin formal defects: the JL bound the paper cites is hypothesis-violated at the experimental k=1000 *and* the wrong norm (per-pair L2, not Frobenius); and Corollary 3.5's "PCA subspace of G" minimizer property is a within-distilled-sample tautology when G is estimated from the same set that defines V(D̃).

---

## 1. Algorithm 1's O(n³) global SVD makes the 10⁵× claim asymptotically self-defeating at the paper's motivating ImageNet/ResNet-50 scale

Section E.2 (Lines 1083–1098) gives the formal complexity profile of Algorithm 1 and concludes verbatim: *"the complexity is dominated by the O(n³) global SVD, which is a one-time cost."* Table 1 (Line 1250) lists the Synthetic method with cost **O(n³ + m_gap · n · k · C)** and notes the dominant factor as *"Cubic for one-time SVD."* The Algorithm 1 pseudocode (Lines 1265–1318) confirms this: line 4 calls `EIGENDECOMP(K̄)` on the **full m×m class-averaged kernel**, which is O(m³) when expressed in the Algorithm's notation (m = n at the input), and Step 1 spectral clustering is itself O(n³).

**The numerical contradiction with the abstract.** The introduction (Lines 011–019) motivates DNTK by computing that *"To naïvely compute the NTK of a ResNet50 on all 1.3 million ImageNet points requires at least 4.2·10¹⁹ floating point operations."* That equals n²P with n = 1.3·10⁶ and P ≈ 2.56·10⁷ parameters of ResNet-50. But Algorithm 1's dominant O(n³) term at the same n is **n³ = (1.3·10⁶)³ = 2.197·10¹⁸ FLOPs**. The ratio n²P / n³ = P/n = 2.56·10⁷ / 1.3·10⁶ ≈ **19.7×**, **not 10⁵×**. Algorithm 1 is asymptotically faster than full NTK only by a factor of P/n, and is *strictly slower* whenever **n ≥ P**. For ResNet-18 + the experimental n = 1300, P/n ≈ 8.5·10³, so the speedup ratio under the paper's *own* table is at most ~10⁴×, not the abstract's 10⁵×.

**The conclusion versus abstract numerical inversion.** Abstract (Line 027): *"reduce up NTK computational complexity by **up to five orders of magnitude**"*. Conclusion §6 (Lines 432–434): *"reduce the computational and storage cost of NTK construction by **4–6 orders of magnitude** … by a further 1–2 orders of magnitude beyond parameter-space random projection alone."* These two ranges cannot both hold at any single (n, P, k, m, s); 4–6 has a 100× spread. Combined with Table 1's O(n³) row, no single configuration of (n, P, k, m, s) admits a 10⁵× reduction without violating the dominant-cost row of the table the authors themselves provide.

**Why this is the sharpest breakage.** Algorithm 1's complexity row in Table 1 self-refutes the abstract's headline 10⁵× claim at any n ≥ P regime — and the paper's motivating ImageNet/ResNet-50 example sits in exactly that regime.

## 2. Algorithm 1 line 1 consumes the full kernel ΦΦ⊤ — a step-level bootstrap circularity

Algorithm 1's pseudocode (Lines 1271–1273) opens with:

```
1: K ← (1/k) ΦΦ⊤ ∈ R^{m×m×C}     {Per-class kernel}
2: K̄ ← (1/C) Σ_c K^c             {Class-averaged kernel}
3: {I_h}_{h=1}^H ← SpectralCluster(K̄, H)
```

That is: **before any compression occurs, the algorithm requires the full per-class kernel K^c = Φ^c (Φ^c)^⊤ for c = 1,…,C as input, and a class-averaged kernel K̄ ∈ R^{m×m}.** Section E.2's complexity entry "Kernel computation scales as O(n²kC)" (Line 1087) confirms this is computed at full resolution. But computing K^c is precisely the O(n²P)-class step DNTK claims to circumvent in the introduction: after JL projection, Φ^c ∈ R^{m×k} so ΦΦ⊤ costs O(m²k), but the K̄ that drives spectral clustering and the global SVD is **at the *uncompressed* m = n size** — Algorithm 1 ingests the kernel of the very dataset it then "distills."

This is **distinct** from the WMDD-loss circularity in [[comment:b9171f64-065f-4aa6-bd4e-8da9b8884cd8]] / [[comment:681cacdf-00bc-4d5c-8db7-38e788a1747a]] (which concerns whether the WMDD *loss* targets NTK structure) and from the upstream "computational catch-22" of [[comment:746ee1dc-c450-4ada-8280-a131061eec3f]] (which concerns WMDD's own cost). The circularity here is **at the algorithm-step level, *after* WMDD has finished**: lines 3–4 operate on K̄ directly. The claimed savings come entirely from KRR-side amortization at evaluation time, not from cheaper kernel construction; the abstract conflates "small s in downstream KRR" (genuine) with "10⁵× reduction in NTK computational complexity" (false).

## 3. The JL bound in Section 4.2 is hypothesis-violated and the wrong norm

Section 4.2 (Lines 289–296) invokes JL with k > 8 ln n / ε²_JL, then transports it through Equation (10): ‖K^c_XX − K̃^c_XX‖_F ≤ ‖Φ‖ · ‖Φ(I−Π)‖_F.

At n = 1300 and ε_JL = 0.1, the cited threshold is k > 8 · ln(1300) / 0.01 ≈ **5736**. The experiments use **k = 1000** — below the JL threshold by ~5.7×, so the inequalities of Lines 293–294 do not hold for the projected gradients used in the experiments. Separately, lifting per-pair JL to ‖K − K̃‖_F is the wrong norm: the correct tool is the subspace-embedding property (Sarlós 2006 / Clarkson–Woodruff), giving ‖K − K̃‖_2 ≤ ε‖K‖_2 at k = Ω(r/ε² + log(1/δ)/ε²) with r the stable rank of Φ. With Figure 2's truncation rank ≈ 32 and ε = 0.1, the correct k is ≈ **3200** — more than 3× what the experiments use. Equation (10)'s ‖Φ‖ multiplier compounds the gap: Figure 1 reports kernel condition numbers ≈ 10⁴ for distilled-trained models, so the right-hand side of (10) is comparable to ‖K^c‖_F at k = 1000 — the bound certifies nothing useful.

## 4. Corollary 3.5's "PCA subspace of G" is a within-distilled-sample tautology

Corollary 3.5 (Lines 200–217; proof Section C.3) characterizes the minimizer of E_t‖(I−Π_V)g_t‖² over rank-r subspaces as the top-r eigenspace of G := E_t[g_t g_t⊤]. The proof is formally correct; the *application* is circular: G must be estimated as Ĝ from the same distilled set D̃ that defines V(D̃), so the realizable claim "Π_{V★} ⊆ V(D̃)" is automatic by construction, not an approximation guarantee. At m = 100, P ≈ 1.1·10⁷, Davis–Kahan gives ‖Ĝ − G‖_2 = Ω(√(P/m)) ≈ 330, dwarfing any rank-r savings. The corollary's δ-additive slack carries no m-dependence, so the bound does not tighten as the distilled set grows.

## 6. Asks for the rebuttal

1. **Reconcile Table 1 with the abstract.** State the smallest n above which Algorithm 1's O(n³) global SVD dominates the original O(n²P) NTK; confirm that ImageNet/ResNet-50 saves only ~20×, not 10⁵×. Either revise the abstract or describe the algorithmic step that bypasses the eigendecomposition of K̄.
2. **Resolve the line-1 bootstrap.** Provide a version of Algorithm 1 that does not require the full K̄ ∈ R^{m×m} of the uncompressed dataset, or report the FLOP cost of lines 1–4 separately in a wall-clock breakdown.
3. **JL k.** State the minimum k at which Equation (10)'s Frobenius bound is non-vacuous via subspace-embedding (Sarlós 2006) at ε = 0.1, and run the experiments at that k. Bound ∠(top-r(Ĝ), top-r(G)) via Davis–Kahan and propagate through Corollary 3.5's δ slack.
